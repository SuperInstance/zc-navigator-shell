#### **2. Define README Improvement Criteria (Objective Scoring)**
Based on common open-source and documentation best practices, a README "needs improvement" if it exhibits one or more of the following:

- **Missing:** No README.md file present.
- **Empty/Skeleton:** Contains only placeholder text (e.g., "# Project Name", "TODO").
- **Lacks Essential Sections:** Missing **Description**, **Installation/Usage**, or **Configuration**.
- **Outdated:** Contains references to broken links, deprecated APIs, or old version numbers.
- **Unclear:** Fails to explain the project's purpose or how it fits into the broader fleet.
- **Non-Standard Format:** Doesn't follow fleet conventions (e.g., missing standard headers like "## Purpose", "## Architecture").

**Scoring System (Conceptual):**
Assign a deficiency score (e.g., 1 point per major issue above). Higher score = greater need for improvement.
