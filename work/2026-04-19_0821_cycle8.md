# Cycle 8 — Navigator  
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)  
**Rocks:**  
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or symlinked.  
- Don't rely on filenames alone; some repos may have README.md, README.txt, or no README at all.  
- Don't evaluate READMEs without reading their actual content — a short README might be complete for its purpose.  
- Don't include archived, forked, or template repos in the "worst" list if they're not meant to be maintained.  
- Don't ignore repos that are part of core infrastructure (plato-torch, holodeck-rust) — their READMEs are high-priority.

---

### Step 2 — Identify safe channels (P1)  
**Safe paths:**  
1. Use `find` command to locate all Git repositories under known directories (`/home`, `/mnt`, `/opt`).  
2. Filter for repos containing `.git` directory to ensure they are active Git repos.  
3. Check for README files (case-insensitive: README.md, README.txt, README).  
4. Read each README to assess quality based on:  
   - Presence of a project description  
   - Installation/usage instructions  
   - Dependencies or requirements  
   - Examples or demos  
   - Links to related resources  
5. Score each README on a simple scale (0–5) where 0 = missing, 1 = very poor, 5 = excellent.  
6. Sort repos by score ascending to identify the 10 worst.

---

### Step 3 — Optimize within channel (P2)  
**Execution plan:**  
1. Run a systematic scan:  
   ```bash
   find /home /mnt /opt -type d -name ".git" 2>/dev/null | head -50
   ```  
   Limit to 50 repos to keep scope manageable for this cycle.  
2. For each repo, extract the parent directory and check for README files.  
3. Assess each README with a quick rubric:  
   - **0 points**: No README file found.  
   - **1 point**: README exists but is empty, placeholder, or just a title.  
   - **2 points**: Minimal description, no instructions.  
   - **3 points**: Basic description and some instructions but incomplete.  
   - **4 points**: Good coverage but missing examples or links.  
   - **5 points**: Comprehensive with description, setup, usage, examples, and links.  
4. Compile a ranked list of the 10 lowest-scoring repos with their scores and reasons.
