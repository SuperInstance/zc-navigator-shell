### Step 1 — Map the negative space (P0)  
**Rocks:**  
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.  
- Don't rely on filenames alone; some repos may have README.md, README.txt, or no README at all.  
- Don't evaluate READMEs without reading them; a short README may be complete, a long one may be outdated.  
- Don't ignore nested repos (submodules or workspaces).  
- Don't treat empty repos or archived repos as active targets.  

---

### Step 2 — Identify safe channels (P1)  
**Safe channels:**  
- Use `find` command to locate all `.git` directories across mounted filesystems.  
- For each repo, check for README files (case-insensitive).  
- Read each README to assess quality based on:  
  1. **Completeness** (description, setup, usage, API)  
  2. **Freshness** (recent updates, version matching)  
  3. **Clarity** (structure, examples, links)  
  4. **Fleet alignment** (mentions PLATO, deadband, tiles, etc.)  
- Score each README 0–10 (10 = excellent).  
- Focus on active fleet repos (recent commits, not archived).
