**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks (what NOT to do):**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mount points.
- Don't rely on filenames alone; a file named `README.md` could be empty or a placeholder.
- Don't evaluate based on subjective "quality" without defined criteria.
- Don't ignore repos that lack a README entirely — they are candidates for the "worst" list.
- Don't process binary files or non-text documents.
- Don't assume the current directory context; need to search systematically.

### Step 2 — Identify safe channels (P1)
**Safe channels (where I CAN operate):**
- Use `find` command to locate all `.git` directories across mounted filesystems.
- For each git repo, check for the presence of `README.md`, `README`, or similar.
- Apply objective criteria: file existence, file size, basic structure (has headings, has content beyond boilerplate).
- Limit search to known fleet directories: `/home`, `/mnt`, `/opt`, `/fleet`.
- Use `wc`, `head`, `grep` to assess content.
- Generate a ranked list based on deficiency scores.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. **Locate repos:** `find /home /mnt /opt /fleet -type d -name '.git' 2>/dev/null`
2. **Extract repo paths:** strip the `/.git` suffix.
3. **Check README variants:** for each repo, test `README.md`, `README`, `README.txt`, `README.rst`.
4. **Score each README:**
   - -10 points if missing entirely.
   - -5 points if size < 100 bytes.
   - -3 points if no Markdown headings (`#`).
   - -2 points if contains "TODO" or "FIXME".
   - -1 point if fewer than 3 lines of non‑whitespace text.
5. **Sort by total deficiency (most negative first).**
6. **Output top 10 worst.**

## Implementation

Running the scan now.

```bash
#!/bin/bash
# find_repos.sh
set -e

# Define search roots
roots="/home /mnt /opt /fleet"

# Temporary files
repo_list=$(mktemp)
output=$(mktemp)

# Find all .git directories
for root in $roots; do
    if [ -d "$root" ]; then
        find "$root" -type d -name '.git' 2>/dev/null | sed 's|/.git$||' >> "$repo_list"
    fi
done

# Score each repo
while IFS= read -r repo; do
    score=0
    readme_found=0
    # Check for README variants
    for f in "README.md" "README" "README.txt" "README.rst"; do
        if [ -f "$repo/$f" ]; then
            readme_found=1
            size=$(stat -c%s "$repo/$f" 2>/dev/null || echo 0)
            if [ "$size" -lt 100 ]; then
                score=$((score - 5))
            fi
            if ! grep -q '^#' "$repo/$f" 2>/dev/null; then
                score=$((score - 3))
            fi
            if grep -q -i "TODO\|FIXME" "$repo/$f" 2>/dev/null; then
                score=$((score - 2))
            fi
            nonblank=$(grep -v '^\s*$' "$repo/$f" | wc -l)
            if [ "$nonblank" -lt 3 ]; then
                score=$((score - 1))
            fi
            break
        fi
    done
    if [ "$readme_found" -eq 0 ]; then
        score=-10
    fi
    echo "$score $repo" >> "$output"
done < "$repo_list"

# Sort and show top 10 worst
sort -n "$output" | head -10

# Cleanup
rm -f "$repo_list" "$output"
```
