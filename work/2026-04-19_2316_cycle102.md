### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mount points.
- Don't rely on file extensions alone (e.g., README, README.md, README.txt).
- Don't evaluate repos I cannot read due to permissions.
- Don't include repos that are intentionally minimal (e.g., placeholder, archived).
- Don't rank without clear, objective criteria.

### Step 2 — Identify safe channels (P1)
**Safe channels:**
- Scan known fleet directories: `/home`, `/mnt`, `/opt` for `SuperInstance/` or `cocapn/` patterns.
- Use `find` command to locate README files recursively.
- Apply scoring criteria based on content quality indicators.
- Limit to top 10 based on score (lowest quality).

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. Locate all README files in SuperInstance-related repos.
2. Score each README based on:
   - Presence (0 if missing, 1 if exists)
   - Length (<100 chars = 0, 100-500 = 1, >500 = 2)
   - Structure (has sections like ## Usage, ## Install = 1, else 0)
   - Examples (code snippets or commands = 1, else 0)
   - Links (internal or external references = 1, else 0)
   - Recency (mentions 2024/2025 = 1, else 0)
   Maximum score = 7. Lower score = worse.
3. Sort by score ascending, take top 10.
4. Output list with repo path, README path, score, and brief reason.
