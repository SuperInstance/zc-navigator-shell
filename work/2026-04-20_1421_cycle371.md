### Recommended Method for Future Scanning
When repo access is available, use this checklist to score READMEs (1=poor, 5=excellent):

1. **Missing README** (score 0) — No README.md, README.txt, or similar.
2. **Placeholder Text** — Contains "TODO," "Coming soon," or generic templates.
3. **Outdated Info** — Mentions old versions, broken links, deprecated APIs.
4. **Poor Structure** — No sections (Installation, Usage, API), wall of text.
5. **No Examples** — Lacks code snippets, configuration examples, or demos.
6. **Lack of Context** — Doesn't link to related repos, dependencies, or fleet docs.
7. **Typos/Grammar** — Obvious errors affecting readability.

**Example Repo Evaluation Template:**
