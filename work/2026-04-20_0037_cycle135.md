### Step 1 — Map the Negative Space (P0)
**Rocks (what NOT to do):**
- Don't assume all repos are under `/home` — some may be in other directories or mounted volumes.
- Don't rely on file extensions alone — READMEs may be named `README`, `README.md`, `README.txt`, or have no extension.
- Don't evaluate READMEs without reading their actual content — file size or presence alone is insufficient.
- Don't ignore nested READMEs (e.g., in `docs/` or `src/`) — they may be the primary documentation for subprojects.
- Don't process binary files or non-text files mistakenly named README.
- Don't assume the fleet-knowledge directory is exhaustive — there may be other repo clusters outside it.
- Don't skip repos that have no README at all — they are candidates for creation, not just improvement.
- Don't violate the Deadband Protocol by jumping to scoring (P2) before establishing safe scanning parameters (P1).

### Step 2 — Identify Safe Channels (P1)
**Safe Channels (where I CAN operate):**
1. Use `find` command with appropriate paths and case-insensitive name matching.
2. Filter for text files using `file` command or MIME type checking.
3. Read the first few lines of each README to assess content quality.
4. Define clear, objective criteria for "needs improvement":
   - File is empty or nearly empty (< 50 bytes).
   - Contains only placeholder text (e.g., "TODO", "Coming soon").
   - Lacks basic sections (Description, Installation, Usage).
   - Is outdated (e.g., references old versions or broken links).
   - Poor formatting (no Markdown, excessive line length).
5. Limit search to directories known to contain fleet repos (starting with `fleet-knowledge/` as a trusted source).
6. Use a scoring system based on the above criteria to rank READMEs objectively.

### Step 3 — Optimize Within Channel (P2)
**Execution Plan:**
1. **Scan:** Recursively find all README files in `fleet-knowledge/` and adjacent suspected repo directories.
2. **Filter:** Exclude non-text files and symlinks.
3. **Evaluate:** Apply criteria to each README, assigning a deficiency score.
4. **Rank:** Sort by score descending to identify the 10 most deficient.
5. **Output:** List each with path, score, and specific deficiencies noted.

**Implementation (bash snippet for context):**
```bash
# Example safe scan command
find fleet-knowledge -type f -iname "README*" -exec file {} \; | grep -E "text|ASCII" | cut -d: -f1 > readme_list.txt
```
