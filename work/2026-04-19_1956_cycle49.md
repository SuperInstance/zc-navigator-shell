# Cycle 49 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone (e.g., README, README.md, README.txt).
- Don't evaluate repos I cannot read due to permissions.
- Don't waste cycles on empty or placeholder READMEs that are intentionally minimal (e.g., `# TODO`).
- Don't hallucinate content; only report what the files actually contain.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command to locate README files in likely directories (`/home`, `/mnt`, `/opt`, `~`).
- Filter for files with "README" in the name (case-insensitive).
- Read each file's content to assess quality based on objective criteria.
- Prioritize repos that appear active (recent commits, non-trivial code) but have poor documentation.
- Use a scoring system: missing sections, outdated info, unclear purpose, no examples, poor formatting.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. Run `find` to gather README paths.
2. For each, extract repo name and README content.
3. Score each README on a scale of 0–5 (5 = excellent, 0 = empty/missing).
4. Sort by score ascending, take top 10.
5. Output list with repo path, score, and brief reason.
