### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone — READMEs may be `.md`, `.txt`, or have no extension.
- Don't modify files — this is a scan only.
- Don't include repos that are intentionally minimal (e.g., placeholder repos).
- Don't rank based on subjective "badness" — use objective criteria.

### Step 2 — Identify safe channels (P1)
**Safe channels:**
- Use `find` command to locate README files in known fleet directories.
- Use `wc` (word count) and `head` to assess content length and structure.
- Check for presence of key sections: Description, Installation, Usage, API.
- Flag files that are empty, nearly empty, or contain only placeholder text.
- Cross-reference with repo activity (git log) to avoid penalizing new repos.

### Step 3 — Optimize within channel (P2)
**Execution path:**
1. Locate README files in `/home` and `/mnt` (common fleet locations).
2. Filter by size (< 100 bytes) and content (placeholder keywords).
3. Sort by deficiency score (size + missing sections).
4. Output top 10 candidates with specific deficiencies.

## Scan Results

**Command used:**
```bash
find /home /mnt -type f -iname "README*" -exec sh -c '
  for f; do
    bytes=$(wc -c < "$f" 2>/dev/null || echo 0)
    lines=$(wc -l < "$f" 2>/dev/null || echo 0)
    has_desc=$(grep -i "description\|overview\|about" "$f" | head -1 | wc -l)
    has_install=$(grep -i "install\|setup\|build" "$f" | head -1 | wc -l)
    has_usage=$(grep -i "usage\|example\|run" "$f" | head -1 | wc -l)
    placeholder=$(grep -i "TODO\|FIXME\|placeholder\|coming soon" "$f" | head -1 | wc -l)
    score=$(( (bytes < 100 ? 10 : 0) + (lines < 3 ? 5 : 0) + (has_desc == 0 ? 3 : 0) + (has_install == 0 ? 2 : 0) + (has_usage == 0 ? 2 : 0) + (placeholder > 0 ? 4 : 0) ))
    echo "$score:$bytes:$lines:$has_desc:$has_install:$has_usage:$placeholder:$f"
  done
' _ {} + | sort -t: -k1,1nr | head -20
```
