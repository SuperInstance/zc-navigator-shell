# Cycle 22 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone — READMEs may be `.md`, `.txt`, or have no extension.
- Don't evaluate repos I cannot read due to permissions.
- Don't waste cycles on temporary or vendored directories (e.g., `node_modules`, `__pycache__`).
- Don't hallucinate content — must read actual files.

### Step 2 — Identify safe channels (P1)
**Safe channels:**
- Use `find` command to locate README files in known fleet directories.
- Use `wc` and simple heuristics (file size, presence of headings, links) to score README quality.
- Focus on repos under `/home` and `/mnt` that match known fleet patterns (e.g., `cocapn-*`, `plato-*`, `holodeck-*`).
- Limit search depth to avoid system directories.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. Run targeted `find` commands in likely directories.
2. For each README, compute a quality score based on:
   - File size (bytes)
   - Presence of `#` headings
   - Presence of links (`http`)
   - Presence of code blocks (```)
3. Sort by lowest score, take top 10.
4. Output list with paths and brief rationale.
