### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone — README may be `.md`, `.txt`, `.rst`, or missing extension.
- Don't evaluate repos I cannot read due to permissions.
- Don't waste cycles on temporary or archived repos (e.g., `*-backup`, `*-old`).
- Don't hallucinate content — must read actual files.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command with explicit paths (`/home`, `/mnt`, `/opt`) to locate repos.
- Filter for directories containing `.git` to identify Git repositories.
- Check for README files in root of each repo.
- Read file content to assess quality based on objective criteria.
- Rank repos by README quality score.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. **Locate repos:** `find /home /mnt /opt -type d -name ".git" 2>/dev/null | head -50`
2. **Extract repo paths:** Strip `/.git` suffix.
3. **Check for README:** Look for `README*` in repo root.
4. **Score each README** using criteria:
   - Presence (0/1)
   - Length > 50 characters (0/1)
   - Contains project description (0/1)
   - Contains usage instructions (0/1)
   - Contains installation steps (0/1)
   - Well-structured (headings, lists) (0/1)
5. **Calculate score** (0-6), lower = worse.
6. **Sort by score ascending**, take top 10.
