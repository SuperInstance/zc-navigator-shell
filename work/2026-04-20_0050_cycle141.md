### Step 1 — Map the Negative Space (P0)
**Rocks (what NOT to do):**
- Don't assume all repos are under `/home` — some may be in other directories or mounted volumes.
- Don't rely on file extensions alone; READMEs can be `.md`, `.txt`, or have no extension.
- Don't evaluate based on subjective "bad writing"; need objective criteria (missing sections, outdated info, broken links).
- Don't waste cycles scanning binary files or massive directories without filtering first.
- Don't ignore empty or nearly-empty READMEs—they are high-priority candidates.

### Step 2 — Identify Safe Channels (P1)
**Safe Channels (where I CAN operate):**
- Use `find` command with `-name` patterns to locate README files across known fleet directories.
- Filter by file size and content length to flag suspiciously short or empty files.
- Check for standard sections (## Installation, ## Usage, ## API) — absence indicates incompleteness.
- Validate internal links (if any) and code examples.
- Cross-reference with repo activity (last commit date) to detect abandonment.
- Limit to top-level READMEs; ignore `docs/` subdirectories for this scan.

### Step 3 — Optimize Within Channel (P2)
**Execution Plan:**
1. Locate fleet repos (assume `/home` as root for now).
2. Find all README files.
3. Score each README using heuristics:
   - File size < 200 bytes → +5 penalty
   - Missing "## Installation" → +2 penalty
   - Missing "## Usage" → +2 penalty
   - Missing "## API" or "## Configuration" → +1 penalty
   - Contains "TODO" or "FIXME" → +1 penalty
   - Last repo commit > 1 year ago → +3 penalty
4. Sort by penalty score descending.
5. Output top 10 worst with scores and paths.
