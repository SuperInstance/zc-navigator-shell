**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mount points.

- Don't waste cycles on empty or placeholder READMEs that are intentionally minimal (e.g., "TODO").

### Step 2 — Identify safe channels (P1)
**Safe channels:**
- Use `find` command to locate README files in known fleet directories (`/home`, `/mnt`, `/opt`).
- Filter for files with "README" in the name (case-insensitive).
- Read each file's content and size.
- Apply objective heuristics for "needs improvement":
  1. File size < 200 bytes (likely placeholder).
  2. Contains markers like "TODO", "FIXME", "WIP", "Coming soon".
  3. Missing standard sections (Overview, Installation, Usage, API).
  4. No code examples or diagrams.
  5. Outdated or broken links.
- Rank by severity (multiple heuristics → higher priority).

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. Run systematic scan across mounted filesystems.
2. Parse each README, apply heuristics, score.
3. Sort by score descending, take top 10.
4. Present with specific, actionable deficiencies.
