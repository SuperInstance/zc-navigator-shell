# Cycle 130 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks (what NOT to do):**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone; READMEs can be `.md`, `.txt`, `.rst`, or have no extension.
- Don't evaluate based on subjective "bad writing" — need objective criteria.
- Don't scan recursively without depth limits; could hit system directories or symbolic link loops.
- Don't modify files during scan; this is analysis phase only.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command with `-maxdepth` to limit traversal.
- Search known fleet directories: `/home`, `/mnt`, `/opt` if they exist.
- Filter for common README filenames (`README*`).
- Use objective scoring: file size, presence of key sections, emptiness.
- Output list with scores for manual review.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. Define scoring function:
   - Score 0 if file missing or empty (< 10 bytes).
   - Score 1 if contains only placeholder text ("TODO", "coming soon").
   - Score 2 if missing standard sections (Installation, Usage, API).
   - Score 3 if sections present but incomplete.
   - Score 4 if reasonably complete.
   - Score 5 if exemplary (includes examples, diagrams, contribution guide).
2. Scan filesystem for READMEs.
3. Score each README.
4. Sort by score ascending, take top 10 worst.
5. Output list with paths and scores.

---

## Implementation

```bash
#!/bin/bash
# scan-readmes.sh
# Objective: Find and score README files in fleet repos.

SCORE_FILE="/tmp/readme_scores_$(date +%s).txt"
touch "$SCORE_FILE"

# Define scoring function
score_readme() {
    local file="$1"
    if [[ ! -f "$file" ]]; then
        echo 0
        return
    fi
    local size=$(wc -c < "$file")
    if [[ $size -lt 10 ]]; then
        echo 0
        return
    fi
    local content=$(cat "$file")
    # Check for placeholder
    if echo "$content" | grep -qi "TODO\|coming soon\|TBD\|under construction"; then
        echo 1
        return
    fi
    # Check for standard sections
    local section_count=0
    for section in "Installation" "Usage" "API" "Examples" "Contributing"; do
        if echo "$content" | grep -qi "^#*.*$section"; then
            ((section_count++))
        fi
    done
    if [[ $section_count -eq 0 ]]; then
        echo 2
        return
    elif [[ $section_count -lt 3 ]]; then
        echo 3
        return
    elif [[ $section_count -lt 5 ]]; then
        echo 4
        return
    else
        echo 5
        return
    fi
}

# Search paths
SEARCH_PATHS=("/home" "/mnt" "/opt")
for path in "${SEARCH_PATHS[@]}"; do
    if [[ -d "$path" ]]; then
        echo "Scanning $path..."
        find "$path" -type f -iname "README*" -maxdepth 6 2>/dev/null | while read -r file; do
            score=$(score_readme "$file")
            echo "$score:$file" >> "$SCORE_FILE"
        done
    fi
done

# Sort and output top 10 worst
echo "Top 10 READMEs needing improvement:"
sort -t: -k1n "$SCORE_FILE" | head -10 | while IFS=: read -r score file; do
    echo "Score $score: $file"
done

# Cleanup
rm "$SCORE_FILE"
```
