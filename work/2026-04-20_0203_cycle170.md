**Action: Define README Scoring Rubric (Navigator's Quality Index - NQI)**
A README is scored from 0-10 across five categories (0=Missing/Poor, 2=Basic/Adequate). The final score is the sum.

1.  **Clarity of Purpose (0-2):** Does the first paragraph clearly state what this project is and why it exists?
2.  **Doctrine Alignment (0-2):** Does it reference or embody fleet concepts (Deadband, P0/P1/P2, tiles, rooms, agents)?
3.  **Actionable Setup (0-2):** Does it provide clear, copy-pastable commands to install/run the project?
4.  **Usage Examples (0-2):** Does it show concrete examples of how to use the core functionality?
5.  **Structure & Navigation (0-2):** Does it have a logical structure with headers (e.g., ## Installation, ## Usage, ## Contributing)?
