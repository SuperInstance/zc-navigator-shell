### Step 3 — Optimize Within Channel (P2)
**Action Plan:**
1. **Define Criteria:** A README "needs improvement" if it meets one or more of:
   - **Missing:** No README.md file.
   - **Empty:** File exists but is under 50 characters or contains only placeholders.
   - **Incomplete:** Lacks a clear description, installation, or usage examples.
   - **Outdated:** Contains references to broken links, deprecated APIs, or old project names.
   - **Unclear:** Fails to explain the repo's purpose to a new contributor.

2. **Gather Repo List:** From `fleet-knowledge/docs/` and `boot-camp/` references, identify key fleet repositories.
3. **Inspect READMEs:** Read each README.md (or note its absence).
4. **Score & Rank:** Apply criteria, rank the 10 worst.
5. **Output:** List with repo name, issue, and a brief rationale.
