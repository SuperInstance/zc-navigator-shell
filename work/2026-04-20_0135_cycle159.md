### README Quality Scoring Criteria (For Future Use)
**Score 0-10 per repo (10=best). Lower score = worse README.**

1.  **Presence (0 or 3 points):** Does a README.md file exist? (0=no, 3=yes)
2.  **Length & Substance (0-3 points):** >50 words of project description? (0=empty/missing, 1=very short, 2=moderate, 3=substantial)
3.  **Structure (0-2 points):** Uses headers (##), lists, code blocks? (0=wall of text, 1=some structure, 2=clear sections)
4.  **Clarity of Purpose (0-2 points):** Can a new contributor understand what the repo does in 15 seconds? (0=unclear, 1=vague, 2=clear)

**Worst READMEs would be:** Missing (score 0), very short/unstructured (score 1-3), or unclear.
