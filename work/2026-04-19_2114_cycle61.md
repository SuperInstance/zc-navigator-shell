# Cycle 61 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone — READMEs may be `.md`, `.txt`, or absent.
- Don't evaluate repos I cannot read due to permissions.
- Don't waste cycles on empty or placeholder READMEs that are intentional (e.g., `TODO.md`).
- Don't hallucinate content — must read actual files.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command to locate README files in known fleet directories.
- Filter for files with "README" in the name, case-insensitive.
- Read each file's content and size.
- Apply scoring heuristics: length, structure, completeness, freshness.
- Rank by score, worst first.
- Limit to top 10.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. Search for READMEs in `/home`, `/mnt`, `/opt`, and current directory.
2. For each found, capture: path, size, last modified, first few lines.
3. Score each README:
   - **Length score:** < 100 bytes = 3 (poor), 100–500 = 2, > 500 = 1.
   - **Structure score:** lacks headers (no `#`) = 2, has headers = 1.
   - **Completeness score:** contains "TODO", "FIXME", "placeholder" = 2, else 1.
   - **Freshness score:** older than 180 days = 2, newer = 1.
   - **Total score:** sum of above (lower is better, higher = worse).
4. Sort by total score descending, pick top 10.
5. Output list with reasons.
