# Cycle 45 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on `find` alone; need to verify each repo is part of the SuperInstance fleet.
- Don't judge READMEs solely by length; some may be short but complete.
- Don't modify files during scan; this is analysis phase only.
- Don't include archived or deprecated repos in "worst" list if they are intentionally minimal.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` with `-type f` and `-name "README.md"` to locate files.
- Filter to likely fleet directories: `/home/*`, `/mnt/*`, `/opt/*`.
- Check each repo's root for a `.git` folder to confirm it's a repo.
- Read each README.md and score against a standard rubric.
- Cross-reference with fleet-knowledge docs for known repo taxonomy.
- Output a ranked list based on objective criteria.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. **Locate READMEs:**  
   ```bash
   find /home /mnt /opt -type f -name "README.md" 2>/dev/null | head -100
   ```
2. **Filter to git repos:** For each README path, check if parent directory contains `.git`.
3. **Score each README** using rubric:
   - **Presence** (1 point): File exists.
   - **Length** (1 point): >50 words.
   - **Structure** (1 point): Has at least one header (e.g., `# Title`).
   - **Clarity** (1 point): Clearly states purpose of repo.
   - **Links** (1 point): Contains at least one link (internal or external).
   - **Code example** (1 point): Includes a code block or usage snippet.
   - **Total score:** 0-6.
4. **Sort by score ascending** to find worst.
5. **List top 10 worst** with paths and scores.
