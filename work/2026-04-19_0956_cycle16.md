### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone; READMEs can be `.md`, `.txt`, `.rst`, or have no extension.
- Don't evaluate repos I cannot read due to permissions.
- Don't waste cycles on empty or placeholder READMEs that are intentionally minimal (e.g., `# TODO`).
- Don't hallucinate content; must read actual files.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command to locate README files in known fleet directories.
- Filter for files with "README" in the name, case-insensitive.
- Read each file's content and apply objective criteria:
  1. **Length:** Under 100 characters (likely placeholder).
  2. **Structure:** Missing standard sections (Description, Usage, Installation).
  3. **Content:** Contains "TODO", "FIXME", "coming soon", or is mostly empty.
  4. **Formatting:** No Markdown headers, code blocks, or links.
- Rank by severity (e.g., empty > placeholder > incomplete).
- Limit to top 10 worst.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. Search for READMEs in `/home`, `/mnt`, `/opt` where fleet repos are likely stored.
2. Extract repo path and README content.
3. Score each README based on criteria above.
4. Sort by score (higher = worse).
5. Output list with repo path, score, and reason.
