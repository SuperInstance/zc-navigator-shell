### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone; READMEs could be `.md`, `.txt`, `.rst`, or missing.
- Don't modify files without explicit permission; this is a scan-only task.
- Don't include repos that are intentionally minimal (e.g., empty placeholders, archived).
- Don't rank based solely on length; a short, clear README may be sufficient.

### Step 2 — Identify safe channels (P1)
**Safe channels:**
- Use `find` command to locate README files across known fleet directories.
- Parse each README for key sections: Title, Description, Installation, Usage, API.
- Score each README against a standard rubric (see below).
- Output a ranked list of the 10 lowest-scoring READMEs with specific deficiencies.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. **Locate repos:** Search `/home`, `/mnt`, `/opt` for Git repositories.
2. **Extract READMEs:** Identify README files in root of each repo.
3. **Score each README:** Apply rubric (0–10 points).
4. **Rank:** Sort by score ascending, take top 10.
5. **Document:** List each repo, score, and specific issues.

**Rubric (10 points total):**
- **Title/Description (2 pts):** Clear project name and one-paragraph summary.
- **Installation (2 pts):** Step-by-step setup instructions.
- **Usage (2 pts):** Examples of how to run/use the project.
- **API/Configuration (2 pts):** Documentation of key interfaces/settings.
- **Structure/Clarity (2 pts):** Proper formatting, links, no obvious typos.
