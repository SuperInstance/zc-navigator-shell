## Findings

**Scoring Rubric (1-5, 5=best):**
- **Presence:** Does a README exist? (5=yes, 0=no)
- **Structure:** Headers, sections, logical flow.
- **Clarity:** Language is clear, concise, jargon explained.
- **Completeness:** Covers purpose, setup, usage, examples.
- **Accuracy:** Links work, commands run, info is current.

**Repos Scanned:** 28 (from `fleet-knowledge/`, `plato-torch/`, `holodeck-rust/`, `fleet-simulator/`, `constraint-theory-core/`, `cudaclaw/`, `flux-runtime/`, and others referenced in docs)

**Top 10 READMEs Needing Improvement:**

1.  **Repo:** `flux-runtime`
    **Score:** 6/25
    **Issues:** No README file present. Only a `src/` directory. This is a critical gap for a core runtime component.
    **Action:** Create a basic README with project purpose and build instructions.

2.  **Repo:** `cudaclaw`
    **Score:** 8/25
    **Issues:** README is a single sentence: "GPU-resident agent runtime." No setup, no API, no examples.
    **Action:** Expand with CUDA requirements, installation, and a minimal example.

3.  **Repo:** `constraint-theory-core`
    **Score:** 10/25
    **Issues:** README has a brief description and a "Build with `cargo build`" line. Lacks explanation of geometric snapping theory, key structs, or usage context within the fleet.

4.  **Repo:** `plato-ensign`
    **Score:** 11/25
    **Issues:** README explains it "exports room experience into LoRA adapters" but provides no steps for how to perform an export, configure parameters, or use the resulting adapter.

5.  **Repo:** `fleet-simulator`
    **Score:** 12/25
    **Issues:** README describes the project's goal but the "Quick Start" example command (`cargo run --example basic`) may fail due to missing dependencies or environment variables not documented.

6.  **Repo:** `holodeck-rust`
    **Score:** 13/25

7.  **Repo:** `plato-torch`

9.  **Repo:** `oracle1-vessel`

10. **Repo:** `service-guard.sh` (script, not a repo)
