### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mount points.
- Don't rely on file extensions alone; READMEs may be `.md`, `.txt`, `.rst`, or have no extension.
- Don't evaluate repos without READMEs as "worst" without context; they may be intentionally minimal or WIP.
- Don't ignore hidden directories (`.git`) or symlinks that could lead to duplicate scanning.
- Don't make subjective "worst" calls without defined criteria.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command with appropriate paths and case-insensitive name matching.
- Define clear, objective criteria for "needs improvement":
  1. **Missing**: No README file present.
  2. **Empty**: README exists but has ≤ 50 characters (effectively empty).
  3. **Placeholder**: Contains only generic text (e.g., "TODO", "Coming soon").
  4. **Outdated**: Mentions old versions, deprecated APIs, or broken links.
  5. **Unstructured**: Lacks standard sections (Overview, Installation, Usage, API).
- Limit search to likely SuperInstance directories: `/home`, `/mnt`, `/opt`, excluding system paths.
- Validate each repo is actually part of the fleet (check for `.git` or known fleet patterns).

### Step 3 — Optimize within channel (P2)
**Execution:**

1. **Find all READMEs** in likely SuperInstance paths:
```bash
find /home /mnt /opt -type f -iname "README*" 2>/dev/null | head -100
```
*(Note: Scanning full filesystems is heavy; limiting to 100 candidates for initial pass.)*

2. **Apply criteria** programmatically where possible, manually review borderline cases.

3. **Rank "worst"** by severity: Missing > Empty/Placeholder > Outdated > Unstructured.
