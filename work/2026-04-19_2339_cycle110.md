- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mount points.

- Don't evaluate repos I cannot access due to permissions.
- Don't waste cycles on empty or placeholder READMEs that are intentionally minimal (e.g., `# TODO`).
- Don't hallucinate content; only report what is actually present in the files.

### Step 2 — Identify safe channels (P1)
**Safe channels:**
- Use `find` command to locate README files in known fleet directories.
- Prioritize repos under `/home` and `/mnt` based on typical fleet structure.
- Read each README file fully to assess quality against fleet standards.
- Score READMEs based on: presence of core sections (Overview, Setup, Usage), clarity, completeness, and maintenance status.
- Focus on repos with existing but poor documentation—ignore completely missing READMEs for now.

### Step 3 — Optimize within channel (P2)
**Execution:**
1. **Scan for READMEs:**  
   ```bash
   find /home /mnt /opt -type f -iname "README*" -path "*/SuperInstance/*" 2>/dev/null | head -50
   ```
   *(Note: Running in simulation—will use sampled file list from fleet-knowledge/)*

2. **Assess each README:**  
   Criteria (0-10 score, lower is worse):
   - **0-2:** Missing or nearly empty (e.g., "TODO").
   - **3-5:** Incomplete, outdated, or confusing.
   - **6-8:** Functional but lacking depth or examples.
   - **9-10:** Clear, comprehensive, with setup/usage examples.

3. **Rank the 10 worst** by score and repo impact.
