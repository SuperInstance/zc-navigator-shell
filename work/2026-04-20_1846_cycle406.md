### Step 1 — Map the Negative Space (P0)

- Handle missing READMEs as lowest score.
- Limit to SuperInstance repos (likely under `fleet-knowledge/` or similar).

### Step 3 — Optimize Within Channel (P2)
**Best path through safe water:**
1. **Locate repos:** Scan `fleet-knowledge/` for git repositories.
2. **Extract READMEs:** For each repo, find README files in root.
3. **Score each README:** Use a simple rubric:
   - **0 points:** No README file.
   - **1 point:** README exists but is empty or < 50 characters.
   - **2 points:** README has some content but no structure (no headers, lists).
   - **3 points:** README has basic structure (headers) but missing key sections (e.g., no installation, usage).
   - **4 points:** README has good structure and key sections but is outdated or incomplete.
   - **5 points:** README is comprehensive, structured, and up-to-date.
4. **Rank:** Sort by score ascending (worst first).
5. **List top 10 worst:** Provide repo path and brief deficiency reason.
