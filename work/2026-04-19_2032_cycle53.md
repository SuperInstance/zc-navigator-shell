**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone (e.g., `README`, `README.md`, `README.txt`).
- Don't modify files directly — this is a scan, not a rewrite.
- Don't include repos that are intentionally minimal (e.g., empty placeholder repos).
- Don't rank based solely on length — a short, clear README is better than a long, confusing one.
- Don't ignore hidden directories (`.git`) or non-standard structures.

**Safe channels:**
- Use `find` command to locate all Git repositories under known fleet directories.
- Parse each repo's root for README files.
- Apply scoring heuristics: presence, structure, clarity, completeness.
- Output a ranked list based on objective criteria.

**Execution path:**

3.  Score and rank.
