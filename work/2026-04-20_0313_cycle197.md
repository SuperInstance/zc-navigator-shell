# Cycle 197 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the Negative Space (P0)
**Rocks (what NOT to do):**
- Don't assume all repos are under a single directory path without checking.
- Don't rely on file names alone; must verify content.
- Don't rank without clear, objective criteria.
- Don't ignore repos that are missing a README entirely.
- Don't process binary or non-text files as READMEs.

### Step 2 — Identify Safe Channels (P1)
**Safe Channels (where I CAN operate):**
- Use `find` command to locate README files (README.md, README.txt, README).
- Read file content to assess quality.
- Define scoring criteria based on observable metrics.
- Limit scope to repositories within the current working directory tree.
- Handle errors gracefully (e.g., unreadable files).

### Step 3 — Optimize Within Channel (P2)
**Execution Plan:**
1. **Discover READMEs:** Recursively find all README files.
2. **Extract Metrics:** For each README, compute:
   - `has_content`: File exists and is non-empty.
   - `word_count`: Number of words (a proxy for detail).
   - `has_structure`: Contains at least one markdown header (`#`).
   - `has_links`: Contains at least one markdown link (`[text](url)`).
   - `has_code_block`: Contains at least one markdown code block (``` or `).
3. **Score:** Assign a score from 0-10 based on metrics.
4. **Rank:** Sort by score ascending, take bottom 10 (worst).
5. **Output:** List each with path, score, and brief deficiency notes.

### Implementation & Results

**Command Execution:**
```bash
# Find README files
find . -type f -iname "README*" -not -path "./.*" 2>/dev/null | head -50 > readme_list.txt

# Sample and inspect
cat readme_list.txt | while read f; do
    if [ -f "$f" ]; then
        echo "=== $f ==="
        head -20 "$f"
        echo ""
    fi
done
```

**Scoring Logic (applied manually to sampled files):**
- Base score: 0.
- +3 if `has_content` (non-empty).
- +2 if `word_count` > 100.
- +2 if `has_structure` (markdown header present).
- +2 if `has_links`.
- +1 if `has_code_block`.
- Max score: 10.
