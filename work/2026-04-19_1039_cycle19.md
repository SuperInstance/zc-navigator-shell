# Cycle 19 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone — READMEs may be `.md`, `.txt`, or have no extension.
- Don't evaluate based on subjective "badness" — need objective criteria.
- Don't waste cycles scanning non-repo directories (like `/proc`, `/sys`).
- Don't ignore hidden directories (`.git`) — they define repos.
- Don't process binary files — READMEs are text.

### Step 2 — Identify safe channels (P1)
**Safe channels:**
- Use `find` command to locate `.git` directories — marks repo roots.
- From each repo root, look for files named `README*` (case-insensitive).
- Read first 10 lines of each README to assess quality.
- Apply objective scoring criteria (see below).
- Limit to top 10 lowest scores.

**Scoring criteria (0-10, lower is worse):**
- **0 points:** File missing.
- **1 point:** File exists but empty (0 bytes).
- **2 points:** Contains only placeholder text ("TODO", "Coming soon").
- **3 points:** Single sentence with no context.
- **4 points:** Basic structure but missing key sections (Installation, Usage).
- **5 points:** Has sections but outdated/incorrect info.
- **6 points:** Adequate but poorly formatted.
- **7 points:** Good but missing examples.
- **8 points:** Comprehensive but verbose.
- **9 points:** Excellent but minor typos.
- **10 points:** Perfect (clear, complete, examples, links).

### Step 3 — Optimize within channel (P2)
**Execution path:**
1. Find all `.git` directories in `/home`, `/mnt`, `/opt` (reasonable search scope).
2. For each, locate README file in parent directory.
3. Score each README using criteria.
4. Sort by score ascending.
5. Output top 10 with repo path and score.

**Command sequence:**
```bash
# Find repo roots
find /home /mnt /opt -type d -name ".git" 2>/dev/null | head -50 > /tmp/repos.txt

# Score each README
while read gitdir; do
    repopath=$(dirname "$gitdir")
    readmefile=$(find "$repopath" -maxdepth 1 -type f -iname "README*" | head -1)
    if [ -z "$readmefile" ]; then
        score=0
    elif [ ! -s "$readmefile" ]; then
        score=1
    elif grep -qi "TODO\|Coming soon\|TBD" "$readmefile" 2>/dev/null && [ $(wc -l < "$readmefile") -lt 3 ]; then
        score=2
    elif [ $(wc -l < "$readmefile") -eq 1 ]; then
        score=3
    elif ! grep -qi "install\|usage\|example" "$readmefile" 2>/dev/null; then
        score=4
    elif grep -qi "outdated\|deprecated\|WIP" "$readmefile" 2>/dev/null; then
        score=5
    elif [ $(wc -l < "$readmefile") -lt 10 ]; then
        score=6
    elif ! grep -q "## " "$readmefile" 2>/dev/null; then
        score=7
    elif grep -q "## " "$readmefile" 2>/dev/null && [ $(wc -l < "$readmefile") -gt 50 ]; then
        score=8
    elif grep -q "example\|demo" "$readmefile" 2>/dev/null && [ $(wc -l < "$readmefile") -gt 30 ]; then
        score=9
    else
        score=10
    fi
    echo "$score $repopath"
done < /tmp/repos.txt | sort -n | head -10
```
