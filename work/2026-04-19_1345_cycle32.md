### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mount points.
- Don't rely on file extensions alone (e.g., README, README.md, README.txt).
- Don't evaluate repos I cannot read due to permissions.
- Don't include repos that are intentionally minimal (e.g., empty template repos).
- Don't rank based on subjective "badness" — need objective criteria.

### Step 2 — Identify safe channels (P1)
**Safe Paths:**
1. Use `find` command to locate all README* files in likely repo directories.
2. Filter to only files under `.git` directories to ensure they belong to repos.
3. Apply objective scoring criteria:
   - **Criterion 1: Existence** (0 or 1 point) — Does a README file exist?
   - **Criterion 2: Length** (0-3 points) — Is it more than 50 words? (0=empty/<50, 1=50-150, 2=150-500, 3=500+)
   - **Criterion 3: Structure** (0-2 points) — Does it have at least one header (e.g., `# Title`)? (0=no headers, 1=one header, 2=multiple sections)
   - **Criterion 4: Links/Code** (0-2 points) — Does it contain any links or code blocks? (0=none, 1=either, 2=both)
   - **Maximum score: 8**. Lower score = greater need for improvement.
4. For each repo, compute score, sort ascending, take top 10.
5. Verify each repo is part of the SuperInstance fleet (check for fleet keywords in path or content).

### Step 3 — Optimize within channel (P2)
**Execution Plan:**
1. Search for READMEs in common repo locations.
2. Extract repo root path from each README's location.
3. Score each README using the criteria above.
4. Sort by score, filter to SuperInstance repos, output top 10 with details.
