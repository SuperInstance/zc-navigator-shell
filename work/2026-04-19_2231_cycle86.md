### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone — READMEs may be `.md`, `.txt`, `.rst`, or have no extension.
- Don't evaluate repos I cannot read due to permissions.
- Don't waste cycles on empty or placeholder READMEs that are intentionally minimal (e.g., `# TODO`).
- Don't hallucinate content — must read actual files from the filesystem.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command to locate README files in known fleet directories (e.g., `/home`, `/mnt`, `/opt`).
- Filter for files with "README" in the name (case-insensitive).
- Read each file's content and apply objective scoring criteria.
- Limit search to a reasonable depth (e.g., 3 levels) to avoid scanning entire filesystem.
- Focus on repos under known fleet paths like `/home/ubuntu/fleet-knowledge`, `/home/ubuntu/plato-torch`, etc.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. Run `find` command to gather README file paths.
2. For each README, extract:
   - Repo name (from parent directory)
   - File size
   - Line count
   - Presence of key sections (Description, Installation, Usage, API, Examples, Contributing, License)
   - Presence of broken links (if any)
   - Readability score (subjective but based on structure)
3. Score each README on a scale of 0-10 (0 = worst, 10 = best).
4. Sort by score ascending and return the 10 lowest scores.
