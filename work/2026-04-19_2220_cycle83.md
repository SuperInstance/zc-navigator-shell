**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone; READMEs can be `.md`, `.txt`, or have no extension.
- Don't modify files without explicit permission; this is a scan-only task.
- Don't include repos that are intentionally minimal (e.g., placeholder repos, archived experiments).
- Don't rank based solely on length; a short, clear README is better than a long, confusing one.

### Step 2 — Identify safe channels (P1)
**Safe Paths:**
- Use `find` command to locate README files across known fleet directories.
- Use `wc` (word count) and `head` to assess content length and structure.
- Check for presence of key sections: Description, Installation, Usage, API, Examples.
- Look for obvious issues: empty files, "TODO" placeholders, outdated information, broken links.
- Cross-reference with repo activity (git log) to identify abandoned but still relevant repos.

### Step 3 — Optimize within channel (P2)
**Execution:**

1. **Locate READMEs:**
   ```bash
   find /home -name "README*" -type f 2>/dev/null | head -50
   find /mnt -name "README*" -type f 2>/dev/null | head -30
   find /opt -name "README*" -type f 2>/dev/null | head -20
   ```

2. **Quick assessment script:**
   ```bash
   for readme in $(find /home -name "README*" -type f 2>/dev/null | head -30); do
     echo "=== $readme ==="
     wc -l "$readme"
     head -5 "$readme"
     echo ""
   done
   ```

3. **Manual review of candidates** based on:
   - Empty or near-empty (< 5 lines)
   - Contains "TODO", "FIXME", "PLACEHOLDER"
   - No clear structure or examples
   - Last updated >1 year ago with no recent commits
