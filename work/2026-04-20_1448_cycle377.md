### 2. Criteria for "Needs Improvement"
A README is considered poor if:
- **Missing:** No README file exists.
- **Skeleton:** Contains only placeholder text (e.g., "# Project Name", "TODO").
- **Incomplete:** Lacks essential sections: Overview, Installation, Usage, Examples, API.
- **Outdated:** Mentions old versions, deprecated dependencies, or inactive features.
- **Unclear:** Poor structure, ambiguous instructions, no code examples.
- **Broken:** Contains dead links or incorrect commands.
