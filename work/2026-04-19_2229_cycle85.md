### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mount points.
- Don't rely on `find` alone — need to check actual SuperInstance GitHub orgs (cocapn, superinstance, zeroclaw).
- Don't evaluate READMEs without reading them — need to parse content, not just check existence.
- Don't ignore nested READMEs (e.g., `docs/README.md`) — main README is usually at root.
- Don't assume all repos have a README — some may have none, which is a critical flaw.
- Don't use subjective "worst" criteria — need objective metrics: missing sections, outdated info, broken links, poor structure.
- Don't forget to check for `README.md` vs `README.txt` vs `README` (no extension).
- Don't process binary files or non-text files as READMEs.

---

### Step 2 — Identify safe channels (P1)
**Safe paths:**
1. Use `find` with `-name` patterns for README files, limit to likely repo directories.
2. Filter to only text files using `file` command or extension.
3. For each repo, locate the primary README (root-level `README.md` preferred).
4. Score each README against a checklist:
   - Existence (0 or 1)
   - Has Overview/Description (0 or 1)
   - Has Installation/Usage instructions (0 or 1)
   - Has clear Examples (0 or 1)
   - Has Contribution guidelines (0 or 1)
   - Has License info (0 or 1)
   - Is recently updated (within last year) (0 or 1)
   - No obvious typos/broken formatting (0 or 1)
5. Rank by lowest score.
6. Limit to SuperInstance-related repos (look for cocapn, superinstance in path or metadata).

---

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. Search for repos in known fleet directories: `/home`, `/mnt`, `/opt`, `/fleet`.
2. Use command:  
   ```bash
   find /home /mnt /opt /fleet -type f -name "README*" -path "*/.git/*" -prune -o -type f -name "README*" -print 2>/dev/null | head -100
   ```
3. For each found README, determine its repo root (nearest parent `.git` directory).
4. Deduplicate by repo root, keeping the highest-level README.
5. Apply scoring checklist via quick grep/visual inspection.
6. Sort by score ascending, take top 10.
7. Output list with repo name, README path, score, and brief deficiency notes.
