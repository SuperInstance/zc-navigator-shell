# Cycle 81 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone — README may be `.md`, `.txt`, or absent.
- Don't modify files without reading them first — could break existing documentation.
- Don't ignore hidden directories (`.git`) or temporary files — they may contain relevant metadata.
- Don't assume English — some READMEs may be in other languages.
- Don't evaluate without a clear rubric — subjective "worst" is not actionable.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command with explicit paths (`/home`, `/mnt`, `/opt`) to locate repos.
- Check for README presence and basic structure (title, description, usage).
- Use word count and formatting as objective metrics.
- Prioritize repos that are actively maintained (recent commits).
- Cross-reference with fleet-knowledge taxonomy to identify critical components.
- Validate findings by reading a sample of each README.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. **Locate repos:** `find /home /mnt /opt -type f -name "README*" -o -name "readme*" | head -100`
2. **Extract metadata:** For each README, capture:
   - Repo path
   - File size (bytes)
   - Word count
   - Has headings? (Markdown `#`)
   - Has code blocks? (```)
   - Last modified date
3. **Score objectively:** Assign penalty points:
   - Missing: +10
   - < 100 words: +5
   - No headings: +3
   - No code/examples: +2
   - > 2 years old: +1
4. **Rank worst 10** by penalty score.
5. **Validate** by reading top 3 to ensure scoring aligns with actual quality.
