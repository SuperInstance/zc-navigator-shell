### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mount points.
- Don't rely on file extensions alone; READMEs can be `.md`, `.txt`, `.rst`, or have no extension.
- Don't evaluate repos without READMEs as "worst" — they are a separate category (missing).
- Don't use subjective criteria like "ugly"; use objective metrics: missing sections, outdated info, broken links, no examples.
- Don't scan recursively into every subdirectory; focus on root-level READMEs for main project documentation.
- Don't assume GitHub-style relative links work locally; check for broken image paths or internal anchors.
- Don't waste cycles on archived, deprecated, or fork repos if they exist.
- Don't ignore the possibility of READMEs in parent directories (like `../README.md` for monorepos).

### Step 2 — Identify safe channels (P1)
**Safe channels:**
- Scan known fleet directories: `/home/ubuntu/fleet-knowledge`, `/home/ubuntu/fleet-repos`, `/mnt/fleet-archive`.
- Use `find` command with case-insensitive name matching: `find /home /mnt /opt -type f -iname "readme*" -o -iname "readme.md" -o -iname "readme.txt" 2>/dev/null`.
- Filter out binary files and symlinks: `-type f` and `file --mime-type` check for text.
- Parse each README with objective metrics:
  1. **Length:** Under 100 words → insufficient detail.
  2. **Sections:** Missing at least two of: Description, Installation, Usage, API, Examples, Contributing, License.
  3. **Freshness:** "Last updated" >2 years ago or no date.
  4. **Links:** Check for broken `http://` or `https://` links (curl -I).
  5. **Code blocks:** No executable examples or snippets.
  6. **Images:** Broken local paths (e.g., `![alt](./missing.png)`).
- Rank by total metric violations (higher = worse).
- Limit to top 10 worst, but note if fewer than 10 exist.

### Step 3 — Optimize within channel (P2)
**Execution:**
1. Run scan:
   ```bash
   find /home /mnt /opt -type f \( -iname "readme*" -o -iname "readme.md" -o -iname "readme.txt" \) 2>/dev/null | head -50 > /tmp/readme_list.txt
   ```
2. For each file, compute score (0-6 violations):
   ```python
   # Pseudocode scoring
   score = 0
   if word_count(text) < 100: score += 1
   if missing_sections(text, ['description','installation','usage']) >= 2: score += 1
   if last_updated(text) > '2022-01-01' or no_date: score += 1
   if broken_links(text) > 0: score += 1
   if no_code_blocks(text): score += 1
   if broken_images(text): score += 1
   ```
3. Sort descending by score, take top 10.
4. Output list with repo path, score, and primary issues.
