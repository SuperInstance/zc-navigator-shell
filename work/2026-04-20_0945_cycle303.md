### Step 3 — Optimize Within Channel (P2)
**Optimal Path:**
1. **Locate Repos:** Scan filesystem for Git repositories.
2. **Extract READMEs:** Identify README.md (or similar) in each repo root.
3. **Score Each README:** Using rubric:
   - **0 points:** No README.
   - **1 point:** README exists but is empty or placeholder (e.g., "TODO").
   - **2 points:** README has some content but lacks structure (no headers, no examples).
   - **3 points:** README is structured but incomplete or unclear.
   - **4 points:** README is good but could be improved (e.g., missing badges, outdated).
   - **5 points:** README is excellent (clear, structured, with examples, badges, links).
4. **Rank:** Sort by score ascending (worst first).
5. **List Top 10:** Provide repo path and brief deficiency note.
