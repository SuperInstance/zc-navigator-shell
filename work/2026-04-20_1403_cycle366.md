# Cycle 366 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the Negative Space (P0)
**Rocks (what NOT to do):**
- Don't assume all repos are under a single directory path without checking.
- Don't rely on file names alone; must verify content.
- Don't rank without clear, objective criteria.
- Don't ignore repos that lack a README entirely (they are candidates).
- Don't process binary or non-text files as READMEs.

### Step 2 — Identify Safe Channels (P1)
**Safe Channels (where I CAN operate):**
- Use `find` commands to locate README files (README.md, README.txt, etc.).
- Use `wc` (word count) and `head` to sample content.
- Define scoring criteria based on observable metrics: file size, presence of key sections, readability.
- Limit to first 50 repos found to bound scope.
- Output a structured list with repo path and score.

### Step 3 — Optimize Within Channel (P2)
**Execution Plan:**
1. Find all candidate README files in fleet-knowledge/ and sibling directories.
2. Score each README using:
   - **Length Score:** `wc -l <file>` (lines). <10 lines = poor.
   - **Content Score:** Check for key markers: `#`, `##`, `Description`, `Install`, `Usage`.
   - **Format Score:** Check if markdown or plain text.
3. Rank by lowest composite score.
4. Output top 10 worst with paths and brief rationale.

---

**Action:**
```bash
# Find README files in likely repo locations
find . -type f -iname "README*" -not -path "./.*" | head -50 > /tmp/readme_list.txt
```

**Scoring Script (run for each file):**
```bash
while IFS= read -r file; do
    lines=$(wc -l < "$file" 2>/dev/null || echo 0)
    has_title=$(grep -c "^# " "$file" 2>/dev/null)
    has_sections=$(grep -c "^## " "$file" 2>/dev/null)
    has_install=$(grep -ci "install" "$file" 2>/dev/null)
    has_usage=$(grep -ci "usage" "$file" 2>/dev/null)
    score=$(( (lines < 10 ? 5 : 0) + (has_title == 0 ? 3 : 0) + (has_sections < 2 ? 2 : 0) + (has_install == 0 ? 2 : 0) + (has_usage == 0 ? 2 : 0) ))
    echo "$score:$file"
done < /tmp/readme_list.txt | sort -n | head -10
```

2. Focus on actual code repositories (e.g., plato-torch, holodeck-rust) which were not found in the search path.
3. Adjust search to start from root of git repositories, not just fleet-knowledge/.
