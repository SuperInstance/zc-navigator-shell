### Step 2 — Identify safe channels (P1)
**Safe channels:**
- Use `find` with `-type f` and `-name` patterns to locate README files.
- Restrict search to known SuperInstance directories: `/home/superinstance`, `/mnt/superinstance`, `/opt/superinstance`.
- Filter out binary files, images, and non‑text files using `file` or `grep -I`.
- Read each README fully before scoring; use a simple rubric (presence, structure, clarity, completeness).
- Output a ranked list based on objective criteria.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. Locate all README‑like files in SuperInstance directories.
2. For each file, compute a quick quality score (0–10) based on:
   - **Presence (2 pts):** File exists and is non‑empty.
   - **Structure (3 pts):** Has at least a title, description, and usage section.
   - **Clarity (3 pts):** Language is clear, not overly technical or vague.
   - **Completeness (2 pts):** Includes installation, examples, or links to related repos.
3. Sort by lowest score, take top 10.
4. Output list with repo path, filename, score, and brief reason.
