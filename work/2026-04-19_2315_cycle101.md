# Cycle 101 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mount points.
- Don't rely on file extensions alone; README may be `.md`, `.txt`, `.rst`, or absent.
- Don't evaluate repos I cannot read due to permissions.
- Don't waste cycles on empty or placeholder READMEs if the repo itself is empty.
- Don't hallucinate content; must read actual files.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command to locate all Git repositories under known fleet directories.
- For each repo, check for README file existence and size.
- Read README content and apply scoring heuristics (length, structure, completeness).
- Rank repos by README quality score (lower = worse).
- Output list of 10 worst with specific deficiencies.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. **Locate repos:** Search `/home`, `/mnt`, `/opt` for `.git` directories.
2. **Extract READMEs:** For each repo, find README* files.
3. **Score each README:**
   - **Existence (0/1):** 0 if missing, 1 if present.
   - **Length (0-3):** 0: <50 chars, 1: 50-200, 2: 200-1000, 3: >1000.
   - **Structure (0-2):** 0: no headers, 1: some headers, 2: clear sections (Overview, Usage, etc.).
   - **Content (0-2):** 0: placeholder/text, 1: basic description, 2: detailed docs with examples.
   - **Freshness (0-1):** 0: last modified >1 year ago, 1: recent.
   - **Total score:** Sum (0-9). Lower = worse.
4. **Rank:** Sort by total score ascending.
5. **Output:** Top 10 worst with repo path, score, and specific deficiencies.
