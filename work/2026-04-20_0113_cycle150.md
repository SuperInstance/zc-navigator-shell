### Step 1 — Map the Negative Space (P0)
**Rocks (what NOT to do):**
- Don't assume all repos are under a single directory path without checking.
- Don't rely on file names alone; must read content to assess quality.
- Don't ignore repos with no README at all (they are candidates).
- Don't rank based on subjective "bad writing"; need objective criteria.
- Don't process non-text files (binaries, images) as READMEs.

### Step 2 — Identify Safe Channels (P1)
**Safe Channels (where I CAN operate):**
- Use `find` command to locate README files (README.md, README.txt, etc.) in fleet-knowledge/ and likely repo directories.
- Read each README's content to assess against criteria.
- Define objective scoring criteria: existence, length, structure, clarity, links, code examples.
- Output a ranked list based on scores.

### Step 3 — Optimize Within Channel (P2)
**Execution Plan:**
1. **Locate READMEs:** Search recursively from current directory for README files.
2. **Scoring Criteria (0-10, lower is worse):**
   - **Existence (0 or 2):** 0 if missing, 2 if present.
   - **Length (0-2):** 0 if empty/<50 chars, 1 if short, 2 if substantial.
   - **Structure (0-2):** 0 if no headers, 1 if some structure, 2 if clear sections.
   - **Clarity (0-2):** 0 if confusing, 1 if basic, 2 if clear purpose.
   - **Links (0-1):** 0 if no links, 1 if has links to docs/code.
   - **Code Examples (0-1):** 0 if none, 1 if includes examples.
3. **Score Each README:** Calculate total (0-10).
4. **Rank:** Sort by score ascending, list top 10 worst.
5. **Output:** List with repo/path, score, and brief reason.
