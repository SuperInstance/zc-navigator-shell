1. Run `find` for README files in likely directories.
2. For each found README, score against criteria.
3. Sort by lowest score, take top 10.
4. Output list with repo path and brief deficiency note.

**Execution:**

```bash

```

**Scoring Criteria (0-10, lower is worse):**
- **0 points:** File missing or empty (< 10 bytes).
- **1 point:** Exists but is a placeholder (e.g., "TODO").
- **2 points:** Contains only a title or one-line description.
- **3 points:** Lacks any structure (no sections, bullet points).
- **4 points:** No code examples or usage instructions.
- **5 points:** No links to related repos or documentation.
- **6 points:** Outdated (mentions old versions or deprecated tools).
- **7 points:** Poor formatting (wall of text, broken markdown).
- **8 points:** Missing installation or setup steps.
- **9 points:** No contribution guidelines or contact info.
- **10 points:** Comprehensive and well-structured.
