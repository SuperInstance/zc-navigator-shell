- Don't skip repos with no README — they are candidates for worst.
- Don't trust timestamps alone — a recent file can be poorly written.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command to locate all Git repositories recursively from likely root directories (`/home`, `/mnt`, `/opt`).

- Read each README fully to assess quality based on standard criteria: clarity, structure, completeness, examples.
- Score READMEs on a simple scale (0–5) where 0 = missing, 1 = very poor, 5 = excellent.
- Focus on SuperInstance-related repos (look for keywords in paths or content: "cocapn", "superinstance", "plato", "fleet").

   ```bash
   find /home /mnt /opt -type d -name ".git" 2>/dev/null | head -50
   ```

2. **Extract repo paths and check READMEs:**  
   For each repo, look for `README*` files in root.

3. **Assess quality criteria:**  
   - **Existence (0–1):** 0 if missing, 1 if present.  
   - **Clarity (0–2):** 0 = confusing, 1 = basic, 2 = clear purpose and usage.

   - **Examples (0–1):** 0 = none, 1 = includes code/command examples.  
   Max score = 5.

4. **Rank and list the 10 worst:**  
   Sort by total score ascending, prioritize missing READMEs.
