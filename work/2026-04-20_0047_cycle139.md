### Step 3 — Optimize Within Channel (P2)
**Execution Plan:**
1. **Locate Repos:** Search for directories containing `.git` subdirectory to identify Git repositories.
2. **Find READMEs:** In each repo root, look for files named `README*` (case-insensitive).
3. **Assess Quality:** Apply a simple scoring rubric:
   - **-10 points:** File missing entirely.
   - **-5 points:** File exists but is empty (0 bytes).
   - **-3 points:** File contains only placeholder text (e.g., "TODO", "Coming soon").
   - **-2 points:** No structure (missing headers, sections).
   - **-1 point:** Lacks basic info (project purpose, setup, usage).
4. **Rank:** Sort repos by total deficiency score (most negative first).
5. **List Top 10:** Output repo path and score, with brief deficiency notes.

**Command Execution:**
```bash
# Find Git repos (assuming common locations)
find /home /opt -type d -name ".git" 2>/dev/null | head -50 | while read gitdir; do
    repopath=$(dirname "$gitdir")
    readmefile=$(find "$repopath" -maxdepth 1 -type f -iname "README*" | head -1)
    if [ -z "$readmefile" ]; then
        score=-10
        echo "$score $repopath # MISSING"
    elif [ ! -s "$readmefile" ]; then
        score=-5
        echo "$score $repopath # EMPTY"
    else
        content=$(cat "$readmefile" 2>/dev/null | head -20)
        score=0
        if echo "$content" | grep -qi "TODO\|Coming soon\|TBD\|^$"; then
            score=$((score - 3))
        fi
        if ! echo "$content" | grep -q "^# "; then
            score=$((score - 2))
        fi
        if ! echo "$content" | grep -qi "purpose\|install\|setup\|usage\|example"; then
            score=$((score - 1))
        fi
        echo "$score $repopath"
    fi
done | sort -n | head -10
```
