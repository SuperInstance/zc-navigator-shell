# Cycle 44 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone — READMEs may be `.md`, `.txt`, or have no extension.
- Don't modify files — this is a scan only.
- Don't include repos that are intentionally minimal (e.g., empty placeholders, archived experiments).
- Don't rank based solely on length — a short, clear README can be sufficient.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command with appropriate paths and case-insensitive name matching.
- Filter for files named `README`, `readme`, or similar.
- Read each file to assess content against a standard checklist.
- Score repos based on objective missing elements.
- Focus on active, non-archived repos in the SuperInstance organization.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. Locate all candidate repos in likely directories.
2. For each repo, locate its README file.
3. Score each README against a standard rubric.
4. Sort by lowest score and list the top 10 needing improvement.

---

## Scan Execution

**Command used:**
```bash
find /home /mnt /opt -type f -iname "README*" -o -iname "readme*" 2>/dev/null | head -50
```

**Sample of found README paths:**
- `/home/ubuntu/fleet-knowledge/docs/README.md`
- `/home/ubuntu/plato-torch/README.md`
- `/mnt/data/superinstance/holodeck-rust/README.md`
- `/opt/fleet-simulator/README.txt`

**Scoring Rubric (0-10, lower is worse):**
- **-2** if file is completely empty or missing.
- **-1** for each missing standard section: Description, Installation, Usage, API, Examples, Contributing, License.
- **-1** if contains only placeholder text (e.g., "TODO", "Coming soon").
- **-0.5** if formatting is broken (e.g., no markdown headers, unclear structure).
- **+1** if includes diagrams, links to other fleet repos, or clear examples.
