# Cycle 96 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone — README may be `.md`, `.txt`, `.rst`, or absent.
- Don't modify files directly — this is a scan, not a rewrite.
- Don't include archived, forked, or empty repos in the "worst" list if they are intentionally minimal.
- Don't trust `ls -la` timestamps as sole indicators of staleness — check git history.
- Don't evaluate private/internal repos that may lack READMEs by design.
- Don't assume English — but for fleet consistency, English READMEs are the standard.

### Step 2 — Identify safe channels (P1)
**Safe channels:**
- Use `find` from known fleet root directories (`/home`, `/mnt`, `/opt`) for `.git` folders.
- Parse `git remote -v` to filter for SuperInstance repos (github.com/Cocapn, github.com/SuperInstance).
- For each repo, check for README existence, length, structure, and last update.
- Score READMEs against fleet standard (PROJECT-VISION.md style: clear thesis, structure, practical examples).
- Rank by deficiency score (missing, empty, outdated, unclear).
- Output list with specific deficiencies and repo paths.

### Step 3 — Optimize within channel (P2)
**Execution path:**
1. Locate fleet repos:
   ```bash
   find /home /mnt /opt -type d -name ".git" 2>/dev/null | head -30
   ```
2. For each `.git` directory, check remote URL for SuperInstance origin.
3. Assess README in repo root:
   - Existence: `README*`
   - Length: word count < 100 → poor
   - Structure: lacks sections (## Purpose, ## Usage, ## API) → poor
   - Freshness: README older than last commit by >6 months → outdated
   - Clarity: subjective but flag if unclear or placeholder text.
4. Score each repo 0–10 (10=worst). Weight: missing=10, empty=8, outdated=6, unstructured=4.
5. Sort by score descending, take top 10.
6. Present list with repo path, score, and specific deficiencies.
