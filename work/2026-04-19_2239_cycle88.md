**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.

- Don't modify files directly — this is a scan, not a rewrite.

- Don't trust timestamps alone — a recent update may still be poor quality.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command to locate README files across known fleet directories.
- Read each README fully to assess structure, clarity, and completeness.
- Score against a standard rubric: presence of sections, examples, links, etc.
- Filter out repos that are archived, deprecated, or external forks.
- Output a ranked list with specific deficiencies noted.

### Step 3 — Optimize within channel (P2)
**Execution:**
1. **Locate repos:** Search `/home`, `/mnt`, `/opt` for Git repos containing README* files.
2. **Extract metadata:** Get repo name, README path, last modified date.
3. **Assess quality:** Apply rubric (see below).
4. **Rank:** Sort by score ascending (worst first).
5. **List top 10:** Include repo, score, and primary issues.

**Rubric (0-10 points):**
- **Title & Description (2 pts):** Clear project name and one-line purpose.
- **Installation (2 pts):** Steps to set up or run.
- **Usage (2 pts):** Example commands or code snippets.
- **Links (2 pts):** Links to related docs, APIs, or other repos.
- **Structure (2 pts):** Sections, headers, consistent formatting.
