### 1. Define README Quality Criteria
A README is assessed on a 0-10 scale (10 = best). Points are deducted for missing elements:

- **(-2) Missing entirely:** No README file found.
- **(-2) Empty or near-empty:** Fewer than 3 lines of meaningful content.
- **(-1) No project description:** Fails to explain what the project is.
- **(-1) No usage instructions:** Lacks basic "how to run" or "how to use".
- **(-1) No installation/setup:** Missing dependencies or build steps.
- **(-1) No link to fleet context:** Doesn't mention Cocapn, PLATO, or related projects.
- **(-1) Poor formatting:** Broken Markdown, lack of headers, code blocks without syntax.
- **(-0.5) Outdated info:** Mentions old versions or deprecated paths.

A perfect README (score 10) includes all elements clearly.
