**Executing scan:**

```bash
#!/bin/bash
# Scan for READMEs in likely SuperInstance directories
BASE_DIRS="/home/user /mnt /opt"
OUTPUT_FILE="/tmp/readme_list.txt"
> "$OUTPUT_FILE"

for dir in $BASE_DIRS; do
    if [ -d "$dir" ]; then
        find "$dir" -type f -name "README*" -o -name "README.md" -o -name "README.txt" 2>/dev/null | while read f; do
            # Check if it's likely a repo (has .git or common project files in parent)
            repo_dir=$(dirname "$f")
            if [ -d "$repo_dir/.git" ] || [ -f "$repo_dir/Cargo.toml" ] || [ -f "$repo_dir/package.json" ] || [ -f "$repo_dir/pyproject.toml" ]; then
                echo "$f" >> "$OUTPUT_FILE"
            fi
        done
    fi
done

echo "Found $(wc -l < "$OUTPUT_FILE") candidate READMEs."
```

**Scoring function (applied manually per README):**
- 0: No README (but file exists as placeholder)
- 1‑3: Minimal, outdated, no examples
- 4‑6: Basic structure, some docs, few links
- 7‑9: Clear, examples, links, updated
- 10: Excellent, with tutorials, API docs, contribution guide
