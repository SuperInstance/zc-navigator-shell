**Safe channels:**

- Read each README's content and assess based on objective criteria: presence of key sections (Description, Installation, Usage), length, clarity, and outdatedness.

### Step 3 — Optimize within channel (P2)
**Execution:**
1. Find READMEs in potential repo locations.
2. Filter to git repos.
3. Score each README.
4. Sort and list the 10 worst.

**Scoring criteria (deficiency points):**
- +1 if README is missing entirely (only a placeholder like `# README`).
- +1 if no Description/Overview section.
- +1 if no Installation instructions.
- +1 if no Usage examples.
- +1 if fewer than 50 words total.
- +1 if contains "TODO" or "FIXME" markers.
- +1 if last modified >1 year ago (outdated).
