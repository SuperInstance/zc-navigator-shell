### 2. Define objective "needs improvement" criteria
A README needs improvement if it exhibits one or more of:
- **Empty or placeholder**: Contains "TODO", "Coming soon", or is mostly blank.
- **Missing critical sections**: No installation, usage, or API examples.
- **Outdated**: References deprecated versions or broken links.
- **Overly terse**: Lacks explanation of purpose or architecture.
- **No examples**: Code snippets absent or non-functional.
- **Poor formatting**: Broken markdown, lack of structure.
