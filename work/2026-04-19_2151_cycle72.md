# Cycle 72 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on `find` alone; need to verify each repo is part of the SuperInstance fleet.
- Don't evaluate READMEs without reading them fully; partial scans lead to false negatives.
- Don't ignore hidden files (`.git`) or non-standard README names (`README.txt`, `Readme.md`).
- Don't modify files during scan; this is analysis only.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` with `-type d -name .git` to locate git repositories.
- Filter to paths containing "SuperInstance" or known fleet prefixes (cocapn-, plato-, holodeck-, etc.).
- Read each `README.md` (or fallback) and score against a checklist.
- Output a ranked list based on deficiency scores.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. Locate all git repos in the filesystem.
2. Filter to likely SuperInstance/fleet repos.
3. For each repo, find and read its README.
4. Score each README against criteria: existence, structure, clarity, examples, links.
5. Rank the 10 lowest scores.

---

## Implementation

```bash
#!/bin/bash
# find_repos.sh
set -e

# Step 1: Find all .git directories (repos)
find /home /mnt /opt -type d -name .git 2>/dev/null | head -100 > /tmp/repo_paths.txt

# Step 2: Filter for SuperInstance/fleet repos
filtered_repos=()
while IFS= read -r gitdir; do
    repopath=$(dirname "$gitdir")
    if [[ "$repopath" == *"SuperInstance"* ]] || \
       [[ "$repopath" == *"cocapn"* ]] || \
       [[ "$repopath" == *"plato"* ]] || \
       [[ "$repopath" == *"holodeck"* ]] || \
       [[ "$repopath" == *"fleet"* ]]; then
        filtered_repos+=("$repopath")
    fi
done < /tmp/repo_paths.txt

# Step 3: Score each README
declare -A scores
for repo in "${filtered_repos[@]}"; do
    score=0
    readme=""
    # Check for common README filenames
    for f in "$repo/README.md" "$repo/README.txt" "$repo/Readme.md" "$repo/README"; do
        if [[ -f "$f" ]]; then
            readme="$f"
            break
        fi
    done
    
    if [[ -z "$readme" ]]; then
        score=$((score + 10)) # No README
        scores["$repo"]=$score
        continue
    fi
    
    # Criteria scoring
    content=$(cat "$readme" 2>/dev/null | head -500)
    if [[ -z "$content" ]]; then
        score=$((score + 9)) # Empty
    else
        # Missing sections
        if ! grep -qi "#" "$readme"; then
            score=$((score + 3)) # No headers
        fi
        if ! grep -qi "install" "$readme"; then
            score=$((score + 2)) # No installation
        fi
        if ! grep -qi "example\|usage" "$readme"; then
            score=$((score + 2)) # No examples
        fi
        if ! grep -qi "license" "$readme"; then
            score=$((score + 1)) # No license
        fi
        if [[ $(wc -l < "$readme") -lt 5 ]]; then
            score=$((score + 4)) # Too short
        fi
    fi
    scores["$repo"]=$score
done

# Step 4: Output top 10 worst
echo "Top 10 repos with poorest READMEs (higher score = worse):"
for repo in "${!scores[@]}"; do
    echo "${scores[$repo]} $repo"
done | sort -nr | head -10
```
