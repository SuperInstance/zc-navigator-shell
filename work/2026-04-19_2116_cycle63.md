### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone — README may be `.md`, `.txt`, `.rst`, or absent.
- Don't modify files directly — this is a scan, not a rewrite.
- Don't include repos that are intentionally minimal (e.g., empty placeholder repos).
- Don't trust last-modified timestamps as sole quality indicator — a recent commit may still have poor content.
- Don't ignore hidden directories (`.git`) — but skip them in the scan.
- Don't assume English — but for fleet standardization, non-English READMEs need translation.
- Don't evaluate binary files — skip images, PDFs, etc.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command to locate README files in known fleet directories.
- Filter by common patterns: `README*`, `readme*`, `Readme*`.
- Check for presence, length, structure (headers, links, examples).
- Score based on: existence, length > 50 chars, has at least one header, has installation or usage instructions.
- Prioritize repos under `/home/ubuntu/fleet-knowledge/` and `/home/ubuntu/fleet-repos/` if they exist.
- Use `wc`, `head`, `grep` for quick analysis without full parsing.
- Output a ranked list with repo path and deficiency reasons.

### Step 3 — Optimize within channel (P2)
**Execution:**
1. Locate fleet repos.
2. For each repo, find README.
3. Apply scoring criteria.
4. Rank by score (lowest = worst).
5. Output top 10 worst with specific issues.
