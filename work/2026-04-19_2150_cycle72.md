# Cycle 72 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on `find` alone — need to verify each repo is part of the SuperInstance fleet.
- Don't rate READMEs without a clear rubric — subjective scoring leads to inconsistent results.
- Don't ignore hidden files (`.git`) — they confirm a directory is a repo.
- Don't process binary files or non-text READMEs (e.g., images, PDFs).
- Don't assume README extension is `.md` — could be `.txt`, `.rst`, or none.
- Don't waste cycles on empty repos or archived projects.
- Don't skip checking for README existence — some repos may lack one entirely.

---

### Step 2 — Identify safe channels (P1)
**Safe paths:**
1. Use `find` with `-type d -name .git` to locate git repositories.
2. Filter to likely SuperInstance paths: `/home/*`, `/mnt/*`, `/opt/*`.
3. For each repo, check for README files (case-insensitive: `README*`, `readme*`).
4. Read first 10 lines of each README to assess quality.
5. Apply a simple rubric:
   - **Missing:** No README file.
   - **Empty:** README exists but < 50 bytes.
   - **Skeleton:** Contains only placeholder text (e.g., "# Project Name", "TODO").
   - **Outdated:** Mentions old versions, broken links, deprecated APIs.
   - **Minimal:** Lacks installation, usage, or contribution guidelines.
6. Rank repos by severity (Missing > Empty > Skeleton > Outdated > Minimal).
7. Limit to top 10 worst.

---

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. Locate repos:
   ```bash
   find /home /mnt /opt -type d -name .git 2>/dev/null | head -30
   ```
2. For each repo, extract parent directory and check README.
3. Score using rubric.
4. Sort and output top 10.

**Implementation:**
```bash
#!/bin/bash
# scan-readmes.sh

RUBRIC_MISSING=5
RUBRIC_EMPTY=4
RUBRIC_SKELETON=3
RUBRIC_OUTDATED=2
RUBRIC_MINIMAL=1

declare -a repo_scores

while IFS= read -r git_dir; do
    repo=$(dirname "$git_dir")
    readme=$(find "$repo" -maxdepth 1 -type f -iname "README*" | head -1)
    
    if [[ -z "$readme" ]]; then
        score=$RUBRIC_MISSING
        reason="Missing README"
    elif [[ $(stat -c%s "$readme" 2>/dev/null || echo 0) -lt 50 ]]; then
        score=$RUBRIC_EMPTY
        reason="Empty or tiny README"
    elif grep -qi "TODO\|FIXME\|PLACEHOLDER\|Coming soon" "$readme" 2>/dev/null && [[ $(wc -l < "$readme") -lt 5 ]]; then
        score=$RUBRIC_SKELETON
        reason="Skeleton/placeholder text"
    elif grep -qi "deprecated\|outdated\|version 0\|WIP" "$readme" 2>/dev/null; then
        score=$RUBRIC_OUTDATED
        reason="Outdated references"
    else
        score=$RUBRIC_MINIMAL
        reason="Minimal but present"
    fi
    
    repo_scores+=("$score:$reason:$repo")
done < <(find /home /mnt /opt -type d -name .git 2>/dev/null | head -30)

# Sort by score (descending) and take top 10
printf "%s\n" "${repo_scores[@]}" | sort -t: -k1,1nr | head -10
```
