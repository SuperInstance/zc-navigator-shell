### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command to locate all directories with `.git` subdirectories.
- Filter to paths likely under SuperInstance ownership (e.g., containing "cocapn", "superinstance", "fleet").
- For each repo, check for README existence and read its content.
- Apply objective scoring criteria: presence, length, structure, clarity, completeness.
- Rank by score ascending to find the worst.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. Find all git repos in the filesystem.
2. For each repo, locate and read README file.
3. Score README on 4 criteria (0-3 points each):
   - **Existence:** 0=missing, 1=empty/<50 chars, 2=minimal, 3=present.
   - **Structure:** 0=no formatting, 1=some headings, 2=sections, 3=full template.
   - **Clarity:** 0=unintelligible, 1=vague, 2=understandable, 3=clear & concise.
   - **Completeness:** 0=no useful info, 1=partial, 2=most key details, 3=comprehensive.
4. Total score 0-12. Lower score = worse README.
5. Sort ascending, take top 10.
