### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on filenames alone; some READMEs may be `.md`, `.rst`, `.txt`, or missing extensions.
- Don't evaluate repos I cannot access due to permissions.
- Don't waste cycles on empty or placeholder READMEs that are intentionally minimal (e.g., `# TODO`).
- Don't hallucinate content; only report what the files actually contain.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command to locate README files in likely directories (`/home`, `/mnt`, `/opt`, current working directory).
- Filter for common README patterns (`README*`, `readme*`).
- Read each file's content and apply objective criteria:
  1. **Length:** Fewer than 5 lines.
  2. **Content:** Placeholder text (`TODO`, `Coming soon`, empty).
  3. **Structure:** Missing standard sections (Description, Installation, Usage).
  4. **Formatting:** No Markdown headers, code blocks, or links.
- Sort by "worst" score (most criteria failed).
- Limit to top 10.

### Step 3 — Optimize within channel (P2)
**Execution:**
1. Run find command to locate README files.
2. For each file, extract:
   - Repo path (inferred from parent `.git` or directory name)
   - File size
   - First 20 lines of content
3. Score each README against criteria.
4. Sort and output top 10.
