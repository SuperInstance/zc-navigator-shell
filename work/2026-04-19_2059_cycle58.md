# Cycle 58 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone — README may be `.md`, `.txt`, `.rst`, or absent.
- Don't modify files directly — this is a scan, not a rewrite.
- Don't include repos that are intentionally minimal (e.g., empty template repos).
- Don't rank based on subjective "badness" — use objective criteria.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command to locate README files across known fleet directories.
- Parse each README for measurable attributes: word count, presence of sections (Install, Usage, API), broken links, outdated timestamps.
- Compare against fleet README standard (from `fleet-knowledge/docs/`).
- Filter out archived, deprecated, or fork repos.
- Generate a scored list based on missing required elements.

### Step 3 — Optimize within channel (P2)
**Execution:**

1. **Locate READMEs:**
```bash
find /home -name "README*" -type f 2>/dev/null | head -50 > /tmp/readme_list.txt
```

2. **Score each README (0-100 scale):**
   - **-20 points** if missing Installation section.
   - **-20 points** if missing Usage/Examples section.
   - **-15 points** if missing API/Configuration.
   - **-10 points** if last update >1 year ago.
   - **-10 points** if word count <100.
   - **-5 points** per broken internal link (if detectable).
   - **+5 points** if includes a "Contributing" section.
   - **+5 points** if includes a "Tests" section.

3. **Apply filters:**
   - Exclude repos with "archive", "deprecated", "fork" in path.
   - Exclude repos under `.git/` or `node_modules/`.

4. **Sort by score (ascending) and output top 10 worst.**
