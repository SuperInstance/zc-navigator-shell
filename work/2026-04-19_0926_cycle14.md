### Step 1 — Map the negative space (P0)
**Rocks:**

- Don't waste cycles on repos that already have good READMEs per fleet standards.

### Step 2 — Identify safe channels (P1)
**Safe Paths:**
1. Use `gh repo list SuperInstance --limit 100` to get the authoritative list of repos.
2. Clone or examine each repo's README.md (or readme.md, README) locally.
3. Score each README against the fleet's standard template (from `fleet-knowledge/docs/`).
4. Focus on repos that are active (recent commits) and have a missing, empty, or severely lacking README.
5. Prioritize repos that are dependencies for other projects (high impact).

### Step 3 — Optimize within channel (P2)
**Execution:**

1. **Fetch repo list:**
   ```bash
   gh repo list SuperInstance --limit 150 --json name,description,updatedAt,isArchived,isTemplate --template '{{range .}}{{.name}} {{.updatedAt}} {{.isArchived}} {{.isTemplate}}{{"\n"}}{{end}}' > /tmp/si_repos.txt
   ```
   *(Note: Assumes `gh` CLI is authenticated and available. If not, fallback to checking local clones.)*

2. **Check local clones** (from `/home/ubuntu/fleet-repos/` and other known locations) for README files.

3. **Score each README** using criteria:
   - **Missing (0 pts):** No README file.
   - **Empty/Skeleton (1 pt):** Fewer than 3 lines or just a title.
   - **Incomplete (2 pts):** Lacks at least two of: description, installation, usage, API, examples.
   - **Adequate (3 pts):** Covers basics but could be clearer or more detailed.
   - **Good (4 pts):** Follows fleet template, includes examples, links, contribution guidelines.

4. **Filter out archived/template repos** and those updated >1 year ago (unless they are core dependencies).

5. **Rank the worst 10** by score (lowest first), considering repo importance.
