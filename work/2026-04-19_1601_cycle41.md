# Cycle 41 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on `find` alone; need to check actual SuperInstance GitHub org.
- Don't evaluate READMEs without a clear rubric (subjective scoring leads to inconsistency).
- Don't waste cycles on repos that are archived, empty, or forks.
- Don't ignore `.md` files that aren't named `README.md` (e.g., `Readme.md`, `README.MD`).

### Step 2 — Identify safe channels (P1)
**Safe Paths:**
1. Use `gh repo list SuperInstance --limit 100` to get all repos from the source.
2. Filter out archived, empty, or forked repos via GitHub API fields.
3. Clone or inspect README content directly from GitHub raw URLs to avoid local path issues.
4. Apply a consistent scoring rubric: presence of key sections (Overview, Installation, Usage, API), clarity, code examples, links.
5. Focus on active repos with recent commits — higher impact for improvement.

### Step 3 — Optimize within channel (P2)
**Execution Plan:**
1. Fetch repo list via GitHub CLI.
2. For each repo, fetch README content and metadata.
3. Score each README on a 0–5 scale:
   - 0: No README
   - 1: Placeholder (e.g., "TODO")
   - 2: Minimal (title only)
   - 3: Basic sections but incomplete
   - 4: Good but missing examples or links
   - 5: Comprehensive with examples, API docs, contribution guidelines
4. Sort by score ascending, pick top 10 worst.
5. Include repo name, score, and specific deficiencies.

---

## Implementation

**Command to fetch repos:**
```bash
gh repo list SuperInstance --limit 100 --json name,isArchived,isFork,updatedAt,defaultBranchRef
```

**Scoring Logic (applied per repo):**
- Check if README exists via `https://raw.githubusercontent.com/SuperInstance/{repo}/{defaultBranch}/README.md`
- Parse content for:
  - Title/header
  - Installation instructions
  - Usage examples
  - API/documentation links
  - Contribution guidelines
  - Code blocks
- Deduct points for missing elements.
