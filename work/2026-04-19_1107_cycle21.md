# Cycle 21 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mount points.
- Don't rely on file extensions alone (e.g., README, README.md, README.txt).
- Don't evaluate repos that are empty, archived, or marked as deprecated without checking first.
- Don't waste cycles on binary files or generated artifacts.
- Don't proceed without verifying the repo actually belongs to the SuperInstance/Cocapn fleet.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command with explicit paths known to contain fleet repos (starting with `/home` and checking common dev mounts).
- Filter for files named `README*` (case-insensitive).
- Check file size and content — exclude empty files or placeholders.
- Validate repo ownership by checking for `.git` directory or known fleet patterns in the repo name/path.
- Score READMEs based on objective, verifiable criteria (length, structure, completeness).

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. Run a targeted `find` to locate README files in likely repo directories.
2. For each README, compute a quality score based on:
   - **Length:** Number of lines (excluding empty lines).
   - **Structure:** Presence of standard sections (e.g., "## Installation", "## Usage", "## API").
   - **Completeness:** Whether it describes the project's purpose and how to use it.
3. Sort by score ascending (worst first).
4. Output the top 10 worst READMEs with their scores and paths.

## Execution & Results

**Command executed:**
```bash
find /home -type f -iname "README*" -path "*/.git/*" -prune -o -type f -iname "README*" -print 2>/dev/null | head -100 | while read f; do
  if [[ -s "$f" ]]; then
    lines=$(grep -v '^[[:space:]]*$' "$f" | wc -l)
    sections=$(grep -c '^##' "$f")
    echo "$lines $sections $f"
  fi
done | sort -n | head -10
```
