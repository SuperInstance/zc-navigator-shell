### B. README Quality Criteria (P0-inspired)
A README needs improvement if it lacks one or more of the following:
*   **C1: Clear Purpose:** Does the first paragraph explain *what* this is and *why* it exists?
*   **C2: Usage Instructions:** Does it contain a "Getting Started" or "Quick Start" section with runnable commands?
*   **C3: API/Interface Definition:** For libraries (`plato-torch`, `cudaclaw`), does it show the core functions/classes?
*   **C4: Integration Context:** Does it explain how it fits into the larger Cocapn fleet (e.g., "This room preset feeds into `plato-ensign`")?
*   **C5: Linkage:** Does it link to related repos, documentation, or issue trackers?
