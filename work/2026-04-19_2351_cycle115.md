# Cycle 115 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file existence without checking — repos may be missing, moved, or empty.
- Don't evaluate READMEs without a clear rubric — subjective scoring leads to inconsistent results.
- Don't process binary files or non-text files as READMEs.
- Don't ignore hidden directories (like `.git`) that shouldn't be scanned.
- Don't assume READMEs are only named `README.md` — could be `README.txt`, `README`, etc.
- Don't waste cycles on extremely large repos without sampling.
- Don't modify files during scan — read-only analysis.

### Step 2 — Identify safe channels (P1)
**Safe channels:**
- Use `find` command with explicit paths (`/home`, `/mnt`, `/opt`) to locate repos.
- Filter for directories containing `.git` to identify Git repositories.
- Look for common README filenames using case-insensitive pattern matching.
- Read first 10KB of each README to assess content without loading entire large files.
- Apply a scoring rubric based on observable metrics (length, structure, freshness, completeness).
- Limit to top-level README files only (ignore nested docs).
- Output a ranked list based on scores, with repo paths and specific deficiencies.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. **Locate repos:**
   ```bash
   find /home /mnt /opt -type d -name ".git" 2>/dev/null | head -100
   ```
   (Limit to 100 repos to avoid timeout; Phase 4 focus is on worst, not all.)

2. **Extract README paths:**
   For each repo dir, check for `README*` at top level.

3. **Score each README (rubric):**
   - **Existence (0/1):** File exists and is readable.
   - **Length (0-3):** 0=empty, 1=<100 chars, 2=100-500 chars, 3=>500 chars.
   - **Structure (0-3):** 0=plain text, 1=has headings, 2=has sections (e.g., Install, Usage), 3=full structure with badges, TOC, examples.
   - **Freshness (0-2):** 0=no date or >2 years old, 1=date within 2 years, 2=updated within last year.
   - **Completeness (0-3):** 0=no useful info, 1=minimal description, 2=includes build/run steps, 3=comprehensive docs with API, examples, troubleshooting.

   Total score = sum (max 12). Lower score = worse README.

4. **Rank bottom 10 by score**, breaking ties by repo name.

5. **Output list** with repo path, score, and specific missing elements.

**Implementation (executed in shell):**

```bash
#!/bin/bash
# Navigator README scanner - Phase 4

RUBRIC() {
    local file="$1"
    local score=0
    # Existence
    if [[ ! -f "$file" ]]; then
        echo "0"
        return
    fi
    score=$((score + 1))
    
    # Length
    local len=$(head -c 10000 "$file" | wc -c)
    if [[ $len -eq 0 ]]; then
        : # score 0
    elif [[ $len -lt 100 ]]; then
        score=$((score + 1))
    elif [[ $len -lt 500 ]]; then
        score=$((score + 2))
    else
        score=$((score + 3))
    fi
    
    # Structure
    if grep -q "^#" "$file" 2>/dev/null; then
        score=$((score + 1))
        if grep -q "Install\|Usage\|Example\|API" "$file" 2>/dev/null; then
            score=$((score + 1))
            if grep -q "\[\!\[.*\]\|Table of Contents" "$file" 2>/dev/null; then
                score=$((score + 1))
            fi
        fi
    fi
    
    # Freshness
    if grep -q "202[5-9]\|202[0-4]" "$file" 2>/dev/null; then
        score=$((score + 1))
        if grep -q "202[5-9]" "$file" 2>/dev/null; then
            score=$((score + 1))
        fi
    fi
    
    # Completeness
    if grep -q "clone\|install\|make\|cargo\|npm" "$file" 2>/dev/null; then
        score=$((score + 1))
        if grep -q "example\|demo\|screenshot" "$file" 2>/dev/null; then
            score=$((score + 1))
            if grep -q "troubleshoot\|faq\|license" "$file" 2>/dev/null; then
                score=$((score + 1))
            fi
        fi
    fi
    
    echo "$score"
}

# Find repos
REPOS=$(find /home /mnt /opt -type d -name ".git" 2>/dev/null | head -100)
RESULTS=""

for repo in $REPOS; do
    repodir=$(dirname "$repo")
    readme=$(find "$repodir" -maxdepth 1 -type f -iname "README*" | head -1)
    if [[ -z "$readme" ]]; then
        score=0
        readme="MISSING"
    else
        score=$(RUBRIC "$readme")
    fi
    RESULTS+="$score|$repodir|$readme"$'\n'
done

echo "SCORE|REPO PATH|README FILE"
echo "-------------------------"
echo "$RESULTS" | sort -t'|' -k1,1n | head -10
```
