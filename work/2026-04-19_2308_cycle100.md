# Cycle 100 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone — READMEs may be `.md`, `.txt`, `.rst`, or have no extension.
- Don't evaluate based on length alone — a short, clear README is better than a long, confusing one.
- Don't ignore hidden directories (`.git`) or temporary files — they are not the target.
- Don't modify files during scan — this is analysis only.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command with explicit paths (`/home`, `/mnt`, `/opt`) to locate repos.
- Filter for `README*` files at repo root.
- Read each README and score based on: clarity, structure, completeness, examples, links.
- Rank by score (lowest = worst).
- Output list with repo path and specific deficiencies.

### Step 3 — Optimize within channel (P2)
**Execution:**
1. **Locate repos:**  
   ```bash
   find /home /mnt /opt -type f -name "README*" -path "*/.git/*" -prune -o -type f -name "README*" -print | head -50
   ```
   (Sample output used for analysis)

2. **Score criteria (1-5, 5=best):**
   - **Clarity:** Is the purpose immediately obvious?
   - **Structure:** Headers, sections, logical flow?
   - **Completeness:** Covers setup, usage, API, examples?
   - **Examples:** Code snippets or commands provided?
   - **Links:** Links to related docs, repos, issues?

3. **Analyze found READMEs:**
