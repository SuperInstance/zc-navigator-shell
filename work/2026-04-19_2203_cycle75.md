### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone (e.g., README, README.md, README.txt).
- Don't evaluate repos I cannot read due to permissions.
- Don't waste cycles on empty or placeholder READMEs that are intentionally minimal.
- Don't hallucinate content; must read actual files.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command to locate README files in known fleet directories.
- Filter for files with size > 0 bytes.
- Read each README and score based on objective criteria:
  1. **Existence** (0 or 1 point): File exists and is readable.
  2. **Length** (0-3 points): >50 words = 3, >20 = 2, >0 = 1, 0 = 0.
  3. **Structure** (0-2 points): Has at least one header (##) = 1, has a list or code block = +1.
  4. **Clarity** (0-2 points): Clear purpose statement = 1, clear usage instructions = +1.
  5. **Freshness** (0-1 point): Mentions 2024/2025 or has recent update = 1.
- Total score: 0-9. Lower score = worse README.
- Focus on `/home` and `/mnt` directories where fleet repos are likely stored.

### Step 3 — Optimize within channel (P2)
**Execution:**
1. Run find command to list README files.
2. For each, compute score.
3. Sort by score ascending.
4. Return top 10 worst with scores and file paths.
