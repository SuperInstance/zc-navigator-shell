## **Output: README Assessment Protocol for Navigator**

**Objective:** Systematically identify repos with READMEs needing improvement.

**P0 – Negative Space (What to Avoid):**
*   Do not assess repos not in the fleet's official index.
*   Do not use subjective, vague criteria like "looks bad."
*   Do not assume a README exists without checking.

**P1 – Safe Channels (Assessment Criteria):**
A README is flagged for improvement if it meets **any** of the following:
1.  **Missing:** No `README.md` file present in repo root.
2.  **Skeleton:** File contains only a project name, a single line, or placeholder text (e.g., "# Project Name", "TODO").
3.  **No Purpose:** Lacks a clear description of what the project does and its role in the fleet.
4.  **No Usage:** Does not explain how to build, install, or run the project.
5.  **No Links:** Does not link to related fleet projects or core documentation.
6.  **Format Violation:** Does not follow the fleet's standard README template (if one is discovered/defined).

**P2 – Optimization (Execution Steps):**
1.  Obtain a list of all repository root directories from the fleet's source control.
2.  For each repo, check for `README.md`.
3.  If missing, flag immediately (Score: 0).
4.  If present, parse and check against criteria 2-6.
5.  Assign a deficiency score (e.g., 1 point per failed criterion 2-6).
6.  Rank repos by deficiency score (highest = worst).
7.  Return the top 10.

**Given the current knowledge constraint,** the list of the "10 worst ones" **cannot be generated**. To complete the task, I require:
*   Access to the filesystem or a listing of the actual SuperInstance repositories.
*   Access to the contents of their `README.md` files.
*   (Optional) The fleet's official README standard template.
