# Cycle 136 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the Negative Space (P0)
**Rocks (what NOT to do):**
- Don't assume all repos are under `/home` — some may be symlinked or in other mount points.
- Don't rely on file size alone to judge README quality — a short README could be complete.
- Don't modify files during scan — this is analysis only.
- Don't include repos that are intentionally minimal (e.g., empty template repos).
- Don't skip hidden directories (like `.git`) — they aren't repos to score.
- Don't assume English — some READMEs may be in other languages but still functional.
- Don't treat missing README as "worst" if the repo is a submodule or deprecated stub.

### Step 2 — Identify Safe Channels (P1)
**Where I CAN operate:**
- Scan the known fleet-knowledge/ directory and referenced repo paths.
- Use `find` with `-name README.md` and `-name README` to locate files.
- Score READMEs based on objective criteria: presence, length, structure, clarity.
- Compare against the fleet's documented standards (from BOOT-CAMP.md).
- Output a ranked list based on deficiency scores.

### Step 3 — Optimize Within Channel (P2)
**Best path through safe water:**

1. **Locate Repositories**
   - Start from `/home` (common root in fleet context).
   - Use `find /home -type f -name "README*" -path "*/.git/*" -prune -o -type f -name "README*" -print` to list READMEs outside `.git`.
   - Capture up to 50 unique repo paths.

2. **Define Scoring Rubric**
   - **0 points** = No README file.
   - **1 point** = README exists but < 5 lines.
   - **2 points** = 5–20 lines, minimal structure.
   - **3 points** = 20+ lines, has sections (e.g., Description, Usage).
   - **4 points** = Follows fleet template, includes examples, links.
   - Deduct 1 point if no installation instructions.
   - Deduct 1 point if no example code/usage.

3. **Score Each README**
   - Read first 100 lines of each file.
   - Apply rubric.
   - Sort by lowest score.

4. **List Top 10 Worst**
   - Include repo path, score, and brief deficiency note.
