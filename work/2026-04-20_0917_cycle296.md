### 1. README Quality Criteria (Based on Fleet Docs)
A good fleet README should have:
- **Clear Purpose:** States the project's role in the fleet (e.g., "This is a MUD server with sentiment-aware NPCs").
- **Doctrine Alignment:** Mentions Deadband Protocol or other fleet principles.
- **Getting Started:** Basic setup/usage instructions.
- **API/Usage Examples:** Code snippets or command examples.
- **Links to Related Repos:** Context within the fleet ecosystem.
- **Current Status:** Indicates if active, experimental, or archived.

Penalties for:
- Missing entirely (-5 points)
- Single sentence or placeholder (-4 points)
- No examples or usage (-3 points)
- No fleet context (-2 points)
- Outdated or incorrect info (-2 points)
- Poor formatting/markdown (-1 point)

Score: 10 = excellent, 0 = poor. "Worst" = lowest score.
