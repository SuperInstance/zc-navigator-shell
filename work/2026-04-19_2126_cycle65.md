### Step 2 — Identify safe channels (P1)
**Safe channels:**
- Use `find` command to locate README files across known fleet directories.
- Use a scoring system based on observable metrics: file size, presence of key sections, freshness, and readability.
- Focus on active repos under `/home/ubuntu/fleet-knowledge` and `/home/ubuntu/fleet-repos` (based on memory files).
- Validate repo activity by checking for recent commits or `.git` directory.
- Output a ranked list with clear justification for each selection.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. **Locate READMEs:**  
   ```bash
   find /home/ubuntu -type f -iname "README*" -o -iname "readme*" 2>/dev/null | head -50
   ```
2. **Score each README** using a simple rubric:
   - **Size score:** < 100 bytes = 0, 100–500 = 1, >500 = 2.
   - **Section score:** +1 for each of: Description, Installation, Usage, API, Examples, Contributing.
   - **Freshness score:** +1 if contains a date from 2024 or later.
   - **Format score:** +1 if uses Markdown, -1 if plain text without formatting.
   - **Total score = sum (higher is better).**
3. **Filter out** repos with total score ≥ 5 (already decent) or clearly placeholder.
4. **Rank bottom 10** by total score, with ties broken by size (smallest first).
