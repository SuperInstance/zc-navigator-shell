### 2. Assess READMEs
**Scoring criteria (1=worst, 5=best):**
- **Existence (E):** 0 if missing, 1 if present.
- **Length (L):** 1 if <100 chars, 3 if 100-500, 5 if >500.
- **Structure (S):** 1 if no headers, 3 if some headers, 5 if clear sections (##, ###).
- **Freshness (F):** 1 if no dates/version, 3 if outdated (>1 year), 5 if recent.
- **Completeness (C):** 1 if no usage examples, 3 if basic examples, 5 if clear setup/run steps.

## Findings & Recommendations

**Critical Gaps:**
- Three vessel repos (`oracle1-vessel`, `jetsonclaw1-vessel`, `forgemaster-vessel`) have **no README**. These are key runtime environments for fleet agents.
- Core tech (`cudaclaw`, `flux-runtime`) have only tagline READMEs, hindering contributor onboarding.
- Several repos lack **versioning dates** and **concrete run commands**.

**Proposed P1 Action (Safe Channel):**
- Create minimal README templates for missing repos.
- Add a "Quick Start" section to underspecified READMEs.
- Standardize header structure across fleet: `## Overview`, `## Setup`, `## Usage`, `## API`.
