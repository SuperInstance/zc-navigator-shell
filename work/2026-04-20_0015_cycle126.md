# Cycle 126 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mount points.
- Don't rely on a single directory listing; need to check multiple known fleet locations.
- Don't evaluate READMEs without reading them fully; a missing file is different from a bad one.
- Don't ignore hidden directories (`.git`) or symlinks that might point to repos.
- Don't waste cycles on non-repo directories (like `node_modules`, `__pycache__`).
- Don't assume file naming (`README`, `README.md`, `README.txt`); check all variants.
- Don't proceed without verifying the repo actually belongs to the SuperInstance fleet (check for `.cocapn` markers, specific orgs).
- Don't rank without clear, objective criteria for "worst."

### Step 2 — Identify safe channels (P1)
**Safe Channels:**
- Use `find` command with `-type d -name .git` to locate all Git repositories in known fleet paths (`/home`, `/mnt`, `/opt`).
- For each repo, check for README files (case-insensitive, common extensions).
- Read the README content and score based on objective criteria:
  1. **Existence** (0 or 1): File present?
  2. **Length** (words): <50 = poor, 50–200 = okay, >200 = good.
  3. **Structure** (0–2): Has sections (e.g., Description, Usage, Install)? 0=no, 1=some, 2=clear.
  4. **Clarity** (0–2): Is the purpose explained? 0=unclear, 1=vague, 2=clear.
  5. **Freshness** (0–1): Last updated within last year? 0=no, 1=yes.
- Sum scores (max 8). Lower score = worse README.
- Filter to only repos under `SuperInstance` GitHub org (or with `superinstance` in remote URL).
- Return top 10 lowest scores.

### Step 3 — Optimize within channel (P2)
**Execution:**
1. Run find command to get repo list:
   ```bash
   find /home /mnt /opt -type d -name .git 2>/dev/null | head -50
   ```
   (Limiting to 50 for speed; fleet likely has fewer active repos.)

2. For each repo dir (parent of `.git`):
   - Check remote URL: `git -C <repo> remote get-url origin 2>/dev/null`.
   - Keep only if URL contains `superinstance` or `SuperInstance`.
   - Locate README: `find <repo> -maxdepth 1 -iname 'readme*' -type f`.
   - If none, score 0.
   - If exists, read file, compute criteria scores.

3. Sort by total score ascending, take top 10.

4. Output list with repo path, score, and brief reason.

**Implementation (bash snippet):**
```bash
#!/bin/bash
repos=$(find /home /mnt /opt -type d -name .git 2>/dev/null | head -50)
worst=()
for repo in $repos; do
    repodir=$(dirname "$repo")
    remote=$(git -C "$repodir" remote get-url origin 2>/dev/null | tr '[:upper:]' '[:lower:]')
    if [[ $remote != *"superinstance"* ]]; then
        continue
    fi
    readme=$(find "$repodir" -maxdepth 1 -iname 'readme*' -type f | head -1)
    score=0
    reason=""
    if [[ -z $readme ]]; then
        score=0
        reason="No README"
    else
        content=$(cat "$readme" 2>/dev/null)
        wordcount=$(echo "$content" | wc -w)
        if [[ $wordcount -lt 50 ]]; then
            ((score+=0))
            reason+="short "
        elif [[ $wordcount -le 200 ]]; then
            ((score+=1))
        else
            ((score+=2))
        fi
        if grep -q -i "description\|usage\|install\|example" "$readme"; then
            ((score+=2))
        elif grep -q -i "# " "$readme"; then
            ((score+=1))
        fi
        if [[ $content == *"this is"* || $content == *"purpose"* || $content == *"overview"* ]]; then
            ((score+=2))
        elif [[ $wordcount -gt 10 ]]; then
            ((score+=1))
        fi
        lastmod=$(stat -c %Y "$readme" 2>/dev/null)
        now=$(date +%s)
        if [[ $((now - lastmod)) -lt 31536000 ]]; then
            ((score+=1))
        fi
    fi
    worst+=("$score:$repodir:$reason")
done
printf "%s\n" "${worst[@]}" | sort -t: -k1n | head -10
```
