# Cycle 119 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on `find` alone; need to check actual SuperInstance GitHub org.
- Don't judge READMEs by length alone; need to assess content quality, clarity, and completeness.
- Don't include archived or deprecated repos in the "worst" list if they are intentionally minimal.
- Don't skip checking for README existence (some repos may have none).

### Step 2 — Identify safe channels (P1)
**Safe channels:**
- Use `gh repo list SuperInstance --limit 100` to get all repos from the org.
- For each repo, fetch the README via `gh repo view SuperInstance/<repo> --json readme`.
- Parse the JSON to extract README content and size.
- Apply scoring criteria: presence, length, structure, clarity, examples, links.
- Filter out archived repos (`archived: true` in JSON).
- Sort by score ascending to get the worst performers.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. Install GitHub CLI (`gh`) if not present.
2. Authenticate with minimal read-only scope.
3. Fetch repo list with metadata (name, archived, defaultBranch).
4. For each non-archived repo, fetch README content.
5. Score each README on:
   - **Existence (0 or 1):** 0 if missing, 1 if present.
   - **Length (0-3):** 0 = empty/<50 chars, 1 = very short, 2 = moderate, 3 = substantial.
   - **Structure (0-2):** 0 = no headers, 1 = some headers, 2 = clear sections.
   - **Clarity (0-2):** 0 = unclear/confusing, 1 = basic, 2 = clear and concise.
   - **Examples (0-2):** 0 = none, 1 = minimal, 2 = good examples/code snippets.
   - **Links (0-1):** 0 = no links to docs/issues, 1 = has relevant links.
   - Total score = sum (max 11).
6. Sort by total score ascending.
7. Output top 10 worst READMEs with repo name, score, and brief reason.

**Implementation (bash script):**
```bash
#!/bin/bash
# fetch_superinstance_readmes.sh

REPO_LIST=$(gh repo list SuperInstance --limit 100 --json name,archived,defaultBranch)
echo "Fetched repo list."

declare -A scores
declare -A reasons

while IFS= read -r repo; do
    name=$(echo "$repo" | jq -r '.name')
    archived=$(echo "$repo" | jq -r '.archived')
    
    if [[ "$archived" == "true" ]]; then
        echo "Skipping archived repo: $name"
        continue
    fi
    
    readme_json=$(gh repo view "SuperInstance/$name" --json readme 2>/dev/null)
    if [[ -z "$readme_json" || "$readme_json" == "null" ]]; then
        scores["$name"]=0
        reasons["$name"]="No README found"
        continue
    fi
    
    content=$(echo "$readme_json" | jq -r '.readme.text // ""')
    if [[ -z "$content" || "$content" == "null" ]]; then
        scores["$name"]=0
        reasons["$name"]="Empty README content"
        continue
    fi
    
    # Scoring
    score=0
    reason_parts=()
    
    # Existence
    score=$((score + 1))
    
    # Length
    len=${#content}
    if (( len < 50 )); then
        reason_parts+=("very short")
    elif (( len < 200 )); then
        score=$((score + 1))
        reason_parts+=("short")
    elif (( len < 1000 )); then
        score=$((score + 2))
        reason_parts+=("moderate length")
    else
        score=$((score + 3))
        reason_parts+=("long")
    fi
    
    # Structure (check for headers)
    if [[ "$content" =~ ^# ]]; then
        score=$((score + 2))
        reason_parts+=("has headers")
    elif [[ "$content" =~ ^## ]]; then
        score=$((score + 2))
        reason_parts+=("has headers")
    else
        reason_parts+=("no headers")
    fi
    
    # Clarity (simple heuristic: check for complete sentences)
    sentence_count=$(echo "$content" | grep -o -E '[.!?] +[A-Z]' | wc -l)
    if (( sentence_count > 5 )); then
        score=$((score + 2))
        reason_parts+=("clear")
    elif (( sentence_count > 0 )); then
        score=$((score + 1))
        reason_parts+=("some clarity")
    else
        reason_parts+=("unclear")
    fi
    
    # Examples (check for code blocks or example keywords)
    if [[ "$content" =~ \`\`\` || "$content" =~ "example" || "$content" =~ "Example" ]]; then
        score=$((score + 2))
        reason_parts+=("has examples")
    else
        reason_parts+=("no examples")
    fi
    
    # Links (check for markdown links or URLs)
    if [[ "$content" =~ \[.*\]\(.*\) || "$content" =~ https?:// ]]; then
        score=$((score + 1))
        reason_parts+=("has links")
    else
        reason_parts+=("no links")
    fi
    
    scores["$name"]=$score
    reasons["$name"]=$(IFS=, ; echo "${reason_parts[*]}")
    
done < <(echo "$REPO_LIST" | jq -c '.[]')

echo "Scoring complete. Top 10 worst READMEs:"
echo ""

count=0
for name in "${!scores[@]}"; do
    echo "$name ${scores[$name]} ${reasons[$name]}"
done | sort -n -k2 | head -10 | while read -r name score reason; do
    count=$((count + 1))
    echo "$count. $name (Score: $score/11) - $reason"
done
```
