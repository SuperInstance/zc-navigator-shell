**Rubric for README Quality (Score 1-5 per category, lower is worse):**
- **Completeness (C):** Has Purpose, Installation, Usage, API/Examples, Contributing, License.
- **Clarity (L):** Clear, concise language; well-structured with headers.
- **Accuracy (A):** Information is current and matches the code.
- **Utility (U):** Provides working examples or quickstart commands.
- **Maintenance (M):** Shows signs of recent updates or activity.

**Overall Score = C + L + A + U + M (5-25 scale). Lower score = worse README.**
