**Safe channels:**

- Limit scope to repos referenced in fleet-knowledge docs (plato-torch, holodeck-rust, etc.).
- Output a ranked list with specific deficiencies for each README.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. Scan `/home` and `/mnt` for Git repos containing README files.
2. Apply scoring rubric (0–10) based on:
   - Presence of core sections (Overview, Installation, Usage, API, Examples, Contributing).

   - Links to related repos or documentation.
   - Last update date (within last 6 months = fresh).
   - File length > 100 lines.
3. Sort by lowest score, take top 10.
4. Document each with specific gaps.
