### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone — README may be `.md`, `.txt`, or absent.
- Don't modify files — this is a scan only.
- Don't include repos that are intentionally minimal (e.g., empty template repos).
- Don't rank based solely on length — a short, clear README is better than a long, confusing one.
- Don't process binary files or non-text READMEs.

### Step 2 — Identify safe channels (P1)
**Safe paths:**
- Use `find` command to locate README files in known fleet directories.
- Parse only text files with `README` in the name (case-insensitive).
- Apply scoring heuristics: presence of key sections, clarity, examples, links.
- Focus on repos under `/home` and `/mnt` based on typical fleet structure.
- Output a ranked list with specific deficiencies noted.

### Step 3 — Optimize within channel (P2)
**Execution plan:**
1. Run recursive find for README files.
2. Filter to text files, exclude binaries.
3. Score each README using a simple rubric.
4. Sort by score ascending (worst first).
5. Output top 10 with repo path and deficiencies.
