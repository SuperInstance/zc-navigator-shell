# Cycle 47 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the negative space (P0)
**Rocks:**
- Don't assume all repos are under `/home` — some may be in `/mnt`, `/opt`, or other mounts.
- Don't rely on file extensions alone — README may be `.md`, `.txt`, or absent.
- Don't modify files — this is a scan only.
- Don't include repos that are intentionally minimal (e.g., empty placeholder repos).
- Don't rank based on subjective "bad writing" — focus on missing critical sections.
- Don't process binary files or non-text files.
- Don't assume git is initialized in every directory.

### Step 2 — Identify safe channels (P1)
**Safe channels:**
- Use `find` command with `-type f` and case-insensitive name matching.
- Limit search to known fleet directories: `/home`, `/mnt`, `/opt`.
- Filter for files named `README*` or `readme*`.
- Read first 10 lines of each file to assess structure.
- Score based on objective missing sections: **Description**, **Installation**, **Usage**, **API**, **Examples**, **Contributing**, **License**.
- Repos with no README get lowest score.
- Sort by score ascending (worst first).
- Output top 10 with path and missing sections.

### Step 3 — Optimize within channel (P2)
**Execution:**
1. Run find command across `/home`, `/mnt`, `/opt`.
2. For each README, check for required sections via grep patterns.
3. Calculate missing section count.
4. Sort and limit to 10 worst.

**Command:**
```bash
find /home /mnt /opt -type f -iname "README*" 2>/dev/null | head -50 | while read f; do
    echo "File: $f"
    missing=0
    for section in "Description\|Overview" "Installation\|Setup\|Getting Started" "Usage\|Examples\|Demo" "API\|Reference" "Contributing\|Development" "License\|LICENSE"; do
        if ! grep -qi "$section" "$f" 2>/dev/null; then
            missing=$((missing + 1))
        fi
    done
    echo "Missing sections: $missing"
    echo "---"
done | sort -t: -k2 -n | head -10
```
