### 2. Define README Improvement Criteria
Objective metrics (score 0-2 each, lower is worse):
- **Existence (E):** 0=No README, 1=Placeholder (e.g., just title), 2=Present.
- **Structure (S):** 0=No sections, 1=Basic sections (e.g., "Usage"), 2=Clear sections (Overview, Install, Use, API).
- **Clarity (C):** 0=Unclear or jargon-heavy, 1=Understandable but vague, 2=Clear, concise, newcomer-friendly.
- **Examples (X):** 0=No examples, 1=Minimal example, 2=Working example with output.
- **Links (L):** 0=No links to related repos/docs, 1=Some links, 2=Well-integrated cross-references.

**Total Score = E + S + C + X + L (0-10). Lower score = greater need for improvement.**
