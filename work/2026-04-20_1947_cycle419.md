# Cycle 419 — Navigator
**Task:** Scan SuperInstance repos for READMEs that need improvement. List the 10 worst ones.

---

### Step 1 — Map the Negative Space (P0)
**Rocks (what NOT to do):**
- Don't assume all repos are under a single directory path without checking.
- Don't rely on file names alone; must verify actual README content.
- Don't rank based on subjective "badness" without defined criteria.
- Don't ignore repos that have no README at all (they are candidates).
- Don't process binary files or non-text files as READMEs.
- Don't skip version-controlled files (e.g., in `.git/`).

### Step 2 — Identify Safe Channels (P1)
**Safe Channels (where I CAN operate):**
- Use `find` command to locate files named `README*` or `readme*` (case-insensitive).
- Filter to files under likely SuperInstance directories (e.g., `~/superinstance/`, `~/fleet-knowledge/`).
- Read first 10 lines of each file to assess quality.
- Apply objective criteria: file size (< 100 bytes), placeholder text ("TODO", "coming soon"), missing sections (no Installation, Usage).
- Cross-reference with fleet-knowledge/docs/ for README standards.
- Output list with repo path and specific deficiencies.

### Step 3 — Optimize Within Channel (P2)
**Execution:**
1.  **Locate READMEs:**
    ```bash
    find ~/superinstance ~/fleet-knowledge -type f -iname "readme*" -not -path "*/.git/*" 2>/dev/null | head -30
    ```
    *(Limiting to 30 for initial scan.)*

2.  **Quick Assessment:** For each found path, check:
    - Existence and size: `wc -c < "$file"`
    - Placeholder flags: `grep -i "todo\|coming soon\|placeholder\|write me" "$file"`
    - Structure check: `head -20 "$file"` for Markdown headers, code blocks.

3.  **Rank Deficiencies:** Score each README (lower = worse):
    - -10 points: File missing or 0 bytes.
    - -5 points: Size < 100 bytes.
    - -3 points: Contains placeholder keywords.
    - -2 points: No `## Installation` or `## Usage` sections.
    - -1 point: No code examples or links.

4.  **Output Top 10 Worst:** List with path, score, and reason.
